# 简化脉冲初始规划算法 V4.2  
## 超同步转移轨道 / 标准转移轨道自适应机动量分配

版本：V4.2  
适用对象：多次拱点脉冲初始规划，用于后续有限推力序贯优化。  
核心原则：**不以推进剂最小化作为脉冲规划主目标，而以变轨次数合理、机动量分配平滑、点火时长可实现、经度窗口满足、有限推力初值稳定为主。**

---

# 1. 算法目的

本算法用于从给定初始轨道生成一组多次脉冲变轨初始解。算法应根据初始轨道远地点高度与同步轨道高度的关系，自动区分两类主要任务：

```text
1. 超同步转移轨道：
   初始轨道远地点高度明显高于同步轨道高度。
   典型变轨策略：n 次远地点点火 + 1 次近地点点火。

2. 标准转移轨道：
   初始轨道远地点高度约等于同步轨道高度。
   典型变轨策略：n 次远地点点火。
```

算法同时支持：

```text
1. 自动推荐变轨次数；
2. 用户手动指定变轨次数；
3. 根据用户指定次数重新分配各次点火机动量；
4. 超同步转移轨道采用“前段远地点均匀分配 + 最后远地点/近地点固定尾段”；
5. 标准转移轨道采用“全部远地点按限定条件分配”；
6. 输出可直接用于有限推力优化的脉冲初始表。
```

---

# 2. 轨道类型定义

## 2.1 基本高度定义

初始轨道远地点半径：

```text
r_a0 = a0 * (1 + e0)
```

初始轨道远地点高度：

```text
h_a0 = r_a0 - Re
```

目标同步轨道半径：

```text
r_sync = a_target
```

目标同步轨道高度：

```text
h_sync = a_target - Re
```

其中：

```text
a0       初始轨道半长轴
e0       初始轨道偏心率
Re       地球赤道半径
a_target 目标同步轨道半长轴
```

---

## 2.2 超同步转移轨道

若：

```text
h_a0 > h_sync + supersync_transfer_margin_km
```

等价于：

```text
r_a0 > a_target + supersync_transfer_margin_km
```

则判定为：

```text
orbit_type = supersynchronous_transfer
```

默认判断裕度：

```text
supersync_transfer_margin_km = 500 km
```

含义：

```text
初始轨道远地点高度明显高于同步轨道高度。
```

此类轨道的推荐变轨结构为：

```text
A, A, ..., A, P
```

即：

```text
n 次远地点点火 + 1 次近地点点火
```

若总变轨次数为 `N`，则：

```text
远地点点火次数 = N - 1
近地点点火次数 = 1
```

---

## 2.3 标准转移轨道

若：

```text
|h_a0 - h_sync| <= standard_transfer_apogee_margin_km
```

等价于：

```text
|r_a0 - a_target| <= standard_transfer_apogee_margin_km
```

则判定为：

```text
orbit_type = standard_transfer
```

默认判断裕度：

```text
standard_transfer_apogee_margin_km = 500 km
```

含义：

```text
初始轨道远地点高度约等于同步轨道高度。
```

此类轨道的推荐变轨结构为：

```text
A, A, ..., A
```

即：

```text
n 次远地点点火
```

若总变轨次数为 `N`，则：

```text
远地点点火次数 = N
近地点点火次数 = 0
```

---

## 2.4 其他转移轨道

若既不满足超同步转移轨道，也不满足标准转移轨道，则进入：

```text
orbit_type = general_transfer
```

可选处理方式：

```text
1. 按标准转移轨道处理：全部远地点点火；
2. 按用户指定点火拱点序列处理；
3. 开启自动拱点模式，根据轨道能量和经度窗口选择 A/P。
```

默认推荐：

```text
general_transfer 使用 standard_transfer 的 n 次远地点策略，
除非用户显式指定 apsis_pattern。
```

# 4. 自动推荐变轨次数

## 4.1 单次最大 Δv 估算

对当前质量 `m`，估算单次最大可实现 Δv。

沉底质量消耗：

```text
mp_set = mdot_set * tau_set
m_after_set = m - mp_set
```

沉底贡献速度增量：

```text
dv_set = Isp_set * g0 * ln(m / m_after_set)
```

主机最大点火时间：

```text
tau_main_max = max_total_burn_time_min * 60 - tau_set
```

主机最大消耗：

```text
mp_main_max = mdot_main_eff * tau_main_max
```

主机最大速度增量：

```text
dv_main_max = c_main_eff * ln(m_after_set / (m_after_set - mp_main_max))
```

单次最大速度增量：

```text
dv_burn_max = dv_set + dv_main_max
```

推荐设计速度增量：

```text
dv_burn_design = min(
    design_dv_per_burn_mps,
    burn_utilization * dv_burn_max
)
```

---

## 4.2 总 Δv 粗估

若用户给出：

```text
total_dv_est_user_mps > 0
```

则：

```text
dv_total_est = total_dv_est_user_mps
```

否则使用解析估算或快速试算。

### 4.2.1 标准转移轨道估算

标准转移轨道的远地点高度约为同步高度，主要任务是：

```text
1. 近地点提升 / 圆化；
2. 倾角修正；
3. 偏心率修正；
4. 经度相位修正。
```

可用远地点处速度变化估算：

```text
v_a0 = sqrt(mu * (2/r_a0 - 1/a0))
v_sync = sqrt(mu / a_target)
di = abs(i0 - i_target)
```

合成估计：

```text
dv_total_est ≈ sqrt(v_a0^2 + v_sync^2 - 2*v_a0*v_sync*cos(di))
              + dv_lon_margin
```

其中：

```text
dv_lon_margin = 20 ~ 100 m/s
```

由经度相位修正裕度决定。

---

### 4.2.2 超同步转移轨道估算

超同步转移轨道的远地点高于同步轨道，主要任务是：

```text
1. 前段远地点逐步调整能量和倾角；
2. 最后远地点固定进入指定超同步尾段；
3. 最后近地点回落到目标同步轨道半长轴；
4. 完成终端偏心率、倾角和经度修正。
```

粗估总 Δv 可由：

```text
dv_total_est = dv_front_est + dv_tail_apogee_est + dv_tail_perigee_est
```

若尾段固定半长轴，则尾段速度增量可在初步方向角下由目标半长轴反求得到。若尚未传播，可先使用经验估计值或用户输入值。

---

## 4.3 推荐次数公式

基础推荐次数：

```text
N_raw = ceil(dv_total_est / dv_burn_design)
```

几何最小次数：

```text
if orbit_type == supersynchronous_transfer:
    N_geom_min = 2      # 至少 1 次远地点 + 1 次近地点
    if supersync_tail_fixed_enabled:
        N_geom_min = max(N_geom_min, supersync_tail_fixed_count)

if orbit_type == standard_transfer:
    N_geom_min = 1      # 至少 1 次远地点

if orbit_type == general_transfer:
    N_geom_min = 1
```

为了保持分配平滑，可配置工程最小次数：

```text
engineering_min_count
```

最终推荐：

```text
N_rec = clamp(
    max(N_raw, N_geom_min, engineering_min_count),
    maneuver_count_min,
    maneuver_count_max
)
```

---

## 4.4 用户指定次数

若：

```text
maneuver_count_user > 0
```

则实际采用：

```text
N = maneuver_count_user
```

并进行校核：

```text
1. 超同步转移轨道：N 必须至少为 2；
2. 标准转移轨道：N 必须至少为 1；
3. 若启用固定尾段，N 必须不小于固定尾段次数；
4. 若估计单次 Δv 超过时长能力，输出警告；
5. 若经度窗口无法找到足够点火点，输出不可行。
```

# 5. 机动量分配策略

# 5.1 超同步转移轨道：n 次远地点 + 1 次近地点

总变轨次数：

```text
N
```

点火模式：

```text
burn 1      : A
burn 2      : A
...
burn N - 1  : A
burn N      : P
```

远地点点火次数：

```text
N_A = N - 1
```

近地点点火次数：

```text
N_P = 1
```

---

## 5.1.1 固定尾段启用时

默认启用：

```text
supersync_tail_fixed_enabled = true
```

固定尾段包括：

```text
burn N - 1：最后一次远地点点火
burn N    ：最后一次近地点点火
```

前段点火次数：

```text
N_front = N - 2
```

若：

```text
N_front > 0
```

则前段总机动量：

```text
S_front = front_dv_total_user_mps
```

若用户未指定，则：

```text
S_front = dv_total_est - dv_tail_apogee_est - dv_tail_perigee_est
```

前段均匀分配：

```text
Δv_k = S_front / N_front,   k = 1..N_front
```

或加权均匀分配：

```text
Δv_k = S_front * w_k / Σw_k
```

默认：

```text
w_k = 1
```

尾段处理：

```text
burn N-1：按固定 a_tail_apogee_plus_fixed_km 反求 Δv
burn N  ：按固定 a_tail_perigee_plus_fixed_km 反求 Δv
```

---

## 5.1.2 固定尾段关闭时

若：

```text
supersync_tail_fixed_enabled = false
```

则所有 `N-1` 次远地点和最后一次近地点均参与机动量分配：

```text
Δv_k = dv_total_est / N
```

但仍保持拱点结构：

```text
A, A, ..., A, P
```

最后一次近地点可保留终端修正量：

```text
Δv_N = terminal_correction_reserve_mps
Δv_k = (dv_total_est - Δv_N) / (N - 1)
```

---

# 5.2 标准转移轨道：n 次远地点

总变轨次数：

```text
N
```

点火模式：

```text
burn 1 : A
burn 2 : A
...
burn N : A
```

即：

```text
全部为远地点点火
```

机动量分配方式：

```text
Δv_k = dv_total_est / N,   k = 1..N
```

如果启用末次修正保留：

```text
Δv_N = standard_terminal_reserve_mps
Δv_k = (dv_total_est - Δv_N) / (N - 1),  k = 1..N-1
```

默认建议：

```text
standard_terminal_reserve_mps = 0
```

即全部远地点点火均匀分配。

每次点火均必须满足：

```text
dv_min_per_burn_mps <= Δv_k <= dv_max_per_burn_mps
```

并满足：

```text
tau_total,k <= max_total_burn_time_min
```

若某次超限，处理顺序为：

```text
1. 自动推荐增加变轨次数；
2. 若用户强制指定次数，则输出 warning；
3. 对 Δv 进行限幅并输出终端误差风险；
4. 提醒用户增大 N 或放宽点火时长。
```

# 5.3 通用转移轨道

通用转移轨道默认按标准转移轨道处理：

```text
n 次远地点
```

若用户指定拱点序列，则按用户序列执行。

# 6. 点火点与 q 序列

## 6.1 q 的定义

`q_k` 表示：

```text
第 k 次点火后，选择第 q_k 个满足经度窗口的目标拱点作为第 k+1 次点火点。
```

例如：

```text
q_k = 3
```

表示第 k 次点火后，寻找第 3 个满足经度窗口的远地点或近地点。

---

## 6.2 自动 q 序列

超同步转移轨道：

```text
前段远地点回归 q 可默认取 3；
最后远地点到近地点为自然 A→P 半圈，不需要 q。
```

默认：

```text
q_AA_default = 3
```

标准转移轨道：

```text
所有点火均为远地点，默认每次取第 q_AA_default 个满足窗口的远地点。
```

若用户指定：

```text
q_sequence_user = [q1, q2, ..., qN-1]
```

则优先采用用户设置。

# 7. 方向角定义与校正

## 7.1 当地水平面方向角

第 k 次脉冲方向：

```text
u_k = cos(alpha_k) * Ehat + sin(alpha_k) * Shat
```

其中：

```text
Ehat = 当地正东方向
Shat = 当地正南方向
```

alpha 为正表示向南分量。

---

## 7.2 超同步转移轨道方向角

推荐初值：

```text
前段远地点 alpha = 8 ~ 16 deg
最后远地点 alpha = 8 ~ 16 deg
最后近地点 alpha = 170 ~ 180 deg 或 -180 ~ -170 deg
```

方向角范围：

```text
前段远地点：[-20, 40] deg
最后远地点：[-20, 40] deg
最后近地点：[-180, 180] deg
```

---

## 7.3 标准转移轨道方向角

标准转移轨道全为远地点点火，方向角用于同时调整：

```text
1. 轨道能量；
2. 偏心率；
3. 倾角；
4. 经度相位。
```

推荐初值：

```text
alpha = 5 ~ 20 deg
```

若主要降倾角：

```text
alpha > 0
```

若主要修正能量：

```text
alpha 接近 0
```

方向角范围：

```text
alpha ∈ [-30, 40] deg
```

---

## 7.4 方向角优化目标

方向角优化不以推进剂最小化为目标，而以终端和约束可行为目标：

```text
min J =
    P_terminal
  + P_longitude
  + P_duration
  + P_uniform
  + P_alpha_smooth
```

# 8. 尾段固定半长轴反求 Δv

当超同步转移轨道启用固定尾段且：

```text
tail_control_mode = fixed_post_a
```

需要在尾段点火时通过目标脉冲后半长轴反求 Δv。

给定点火前状态：

```text
r, v
```

给定方向：

```text
u = cos(alpha) * Ehat + sin(alpha) * Shat
```

目标脉冲后半长轴：

```text
a_plus_target
```

要求：

```text
v_plus = v + Δv * u
```

满足：

```text
|v_plus|^2 / 2 - μ / r = -μ / (2 * a_plus_target)
```

展开：

```text
Δv^2 + 2(v·u)Δv + [|v|^2 - 2μ/r + μ/a_plus_target] = 0
```

定义：

```text
B = v·u
C = |v|^2 - 2μ/r + μ/a_plus_target
D = B^2 - C
```

则：

```text
Δv = -B ± sqrt(D)
```

取根规则：

```text
1. D >= 0；
2. Δv > 0；
3. 选择最小正根；
4. 若无正根，则当前 alpha 或状态不可行。
```

# 9. 约束检查

## 9.1 点火经度窗口

每次点火点经度：

```text
lambda_k = sub_satellite_longitude(t_k)
```

脉冲规划阶段建议使用：

```text
lon_window_planning_degE = [45, 175]
```

硬约束窗口：

```text
lon_window_raw_degE = [40, 180]
```

---

## 9.2 点火时长

对每次 Δv，计算主机时长和总点火时长：

```text
mp_set = mdot_set * tau_set
m_after_set = m_before - mp_set

dv_set = Isp_set * g0 * ln(m_before / m_after_set)
dv_main = max(0, dv_impulse - dv_set)

mp_main = m_after_set * [1 - exp(-dv_main / c_main_eff)]
tau_main = mp_main / mdot_main_eff
```

若点火时长包含沉底：

```text
tau_total = tau_set + tau_main
```

否则：

```text
tau_total = tau_main
```

约束：

```text
tau_total <= max_total_burn_time_min * 60
```

---

## 9.3 机动量均匀性

对参与均匀分配的点火集合 `K_uniform`：

```text
dv_bar = mean(Δv_k), k ∈ K_uniform
```

要求：

```text
max(|Δv_k - dv_bar|) <= max_uniform_dv_spread_mps
```

超同步转移轨道中，固定尾段不参与前段均匀性检查。

标准转移轨道中，全部 N 次远地点点火均参与均匀性检查。

---

## 9.4 终端轨道约束

终端轨道参数：

```text
a_f, e_f, i_f, lon_nom_f
```

误差：

```text
da   = a_f - a_target
de   = e_f - e_target
di   = i_f - i_target
dlon = wrap180(lon_nom_f - lon_target)
```

默认容差：

```text
|da|   <= 1 km
|de|   <= 1e-4
|di|   <= 0.01 deg
|dlon| <= 0.05 deg
```

# 10. 目标函数

本算法不把推进剂作为优化目标。推进剂仅作为输出统计。

目标函数：

```text
J =
    P_terminal
  + P_longitude
  + P_duration
  + P_uniform
```

# 11. 超同步转移轨道算法流程

```text
function plan_supersynchronous_transfer(config):

    classify orbit as supersynchronous_transfer

    estimate dv_total_est

    determine N:
        if user sets maneuver_count:
            N = user value
        else:
            N = recommended value

    require:
        N >= 2

    set apsis pattern:
        burn 1..N-1 = A
        burn N      = P

    if supersync_tail_fixed_enabled:
        require N >= 2

        front_count = N - 2

        estimate or read fixed tail:
            burn N-1: fixed apogee tail
            burn N:   fixed perigee tail

        if front_count > 0:
            S_front = estimate_front_total_dv()
            distribute:
                dv_1..dv_front_count = S_front / front_count

        burn N-1:
            solve dv from fixed a_tail_apogee_plus

        burn N:
            solve dv from fixed a_tail_perigee_plus

    else:
        distribute all N burns:
            dv_k = dv_total_est / N

    find burn points by q sequence and longitude window

    optimize alpha and optional small dv corrections:
        no propellant minimization
        minimize terminal and constraint penalties

    output burn table and checks
```

# 12. 标准转移轨道算法流程

```text
function plan_standard_transfer(config):

    classify orbit as standard_transfer

    estimate dv_total_est

    determine N:
        if user sets maneuver_count:
            N = user value
        else:
            N = recommended value

    require:
        N >= 1

    set apsis pattern:
        burn 1..N = A

    distribute dv:
        if standard_terminal_reserve_mps > 0 and N > 1:
            dv_N = standard_terminal_reserve_mps
            dv_1..dv_N-1 = (dv_total_est - dv_N) / (N - 1)
        else:
            dv_k = dv_total_est / N

    check duration:
        each tau_total_k <= limit

    find N apogee burn points satisfying longitude window

    optimize alpha and optional small dv corrections:
        no propellant minimization
        minimize terminal and constraint penalties

    output burn table and checks
```

# 13. 主控流程伪代码

```text
function simplified_impulse_planner_v4_2(config):

    load initial state
    load target
    load engine and constraint parameters

    orbit_type = classify_orbit_by_apogee_height(
        h_a0,
        h_sync,
        supersync_transfer_margin_km,
        standard_transfer_apogee_margin_km
    )

    dv_total_est = estimate_total_dv(orbit_type, initial_orbit, target)

    dv_burn_design = estimate_design_single_burn_dv(engine, burn_limit, m0)

    recommended_count = recommend_maneuver_count(
        dv_total_est,
        dv_burn_design,
        orbit_type
    )

    if maneuver_count_user > 0:
        N = maneuver_count_user
        check_user_count_feasibility(N, orbit_type, recommended_count)
    else:
        N = recommended_count

    if orbit_type == supersynchronous_transfer:
        plan = plan_supersynchronous_transfer(config, N)
    elif orbit_type == standard_transfer:
        plan = plan_standard_transfer(config, N)
    else:
        plan = plan_general_transfer(config, N)

    result = optimize_alpha_and_optional_small_dv(plan)

    propagate result with J2
    compute burn table
    compute propellant statistics
    compute terminal error
    compute constraint checks

    return result
```

# 14. 有限推力初值转换

脉冲时刻应作为主发动机点火弧段中心：

```text
T_impulse,k = 主发动机点火弧段中心
```

主机点火时长由脉冲 Δv 估算：

```text
tau_main,k
```

则有限推力初值：

```text
ts,k = T_impulse,k - tau_main,k / 2
te,k = T_impulse,k + tau_main,k / 2
tb,k = ts,k - tau_set
```

其中：

```text
tb,k = 沉底开始
ts,k = 主机开始
te,k = 主机结束
```

不要将脉冲时刻直接作为沉底开始时刻。

# 17. 结论

V4.2 将轨道分类和策略名称修正为：

```text
1. 超同步转移轨道：
   初始远地点高度明显高于同步轨道高度；
   策略为 n 次远地点点火 + 1 次近地点点火；
   默认最后远地点和最后近地点作为固定尾段。

2. 标准转移轨道：
   初始远地点高度约等于同步轨道高度；
   策略为 n 次远地点点火；
   不启用固定尾段，所有远地点点火按限定条件分配机动量。
```

本算法保留自动推荐变轨次数和用户指定次数能力。用户指定次数时，算法按该次数重新生成点火结构并分配各次机动量；若不满足点火时长或经度窗口约束，则输出明确警告或不可行标志。
