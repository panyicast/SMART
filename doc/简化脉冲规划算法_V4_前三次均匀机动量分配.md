# 简化脉冲规划算法 V4
## 前三次机动量适当均匀分配 + 第四/第五次固定半长轴控制

版本：V4.0  
适用对象：五次拱点脉冲初始规划，用于后续有限推力序贯优化初值生成。  
核心变化：**不再以推进剂最小化作为前三次机动量分配的主优化目标**，而是采用“满足回归、经度、时长、终端约束前提下，前三次机动量适当均匀分配”的工程化策略。

---

# 1. 文档目的

本文档定义一版简化脉冲规划算法，用于替代前一版“以前三次推进剂或总 Δv 微小差异为优化目标”的复杂搜索策略。

前期扫描表明，在第 4、5 次半长轴控制量固定的条件下，前三次机动量在较宽范围内重新分配，对最终推进剂消耗影响很小；但对最大点火时长、有限推力实现裕度、经度窗口裕度影响明显。因此，本版算法不再把推进剂作为主要优化目标，而将目标改为：

```text
1. 满足 3 圈回归与经度窗口；
2. 满足最长点火时间不超过 90 min；
3. 满足第 4、5 次固定半长轴控制；
4. 满足终端轨道 a/e/i/λ 约束；
5. 前三次 Δv 分配尽量均匀，避免某一次点火过大或过小；
6. 输出稳定、平滑、便于转入有限推力优化的初值。
```

---

# 2. 总体思想

## 2.1 原复杂策略

原策略通常写成：

```text
min J = 总推进剂 + 终端误差罚函数 + 约束罚函数
```

或者：

```text
min J = Δv_total + terminal_penalty
```

这种方法能得到数值上略优的脉冲解，但容易出现以下问题：

```text
1. 某一次点火量偏大，接近 90 min 点火时长上限；
2. 某一次点火量偏小，后续点火承担过多补偿；
3. 推进剂收益很小，但有限推力展开裕度变差；
4. 搜索变量较多，计算时间增加；
5. 结果对优化器初值和罚函数权重较敏感。
```

## 2.2 新简化策略

新策略采用：

```text
固定结构 + 均匀分配 + 低维校正
```

即：

```text
第 1、2、3 次：
    以近似均匀的 Δv 分配为主；
    只做必要的方向角和少量机动量修正。

第 4 次：
    固定脉冲后半长轴 a4+；
    作为超同步尾段成形控制。

第 5 次：
    固定脉冲后半长轴 a5+ = 42164.2 km；
    完成最终回落 / 圆化。
```

本版算法的主目标不是“最省推进剂”，而是：

```text
在终端可行的前提下，使前三次点火 Δv 分布平滑、点火时长不过界、有限推力初值更稳健。
```

# 4. 固定策略配置

## 4.1 点火拱点结构

采用五次拱点点火：

```text
1: A  远地点
2: A  远地点
3: A  远地点
4: A  远地点
5: P  近地点
```

其中前四次远地点点火用于升轨、降倾角和建立超同步轨道，第五次近地点点火用于从超同步轨道回落到同步目标半长轴。

## 4.2 回归圈数设置

为简化规划，不再大范围枚举 q 序列，而采用固定回归结构。

推荐默认：

```text
q12 = 3
q23 = 3
q34 = 2
```

即：

```text
第 1 次点火后，取第 3 个可用远地点作为第 2 次点火；
第 2 次点火后，取第 3 个可用远地点作为第 3 次点火；
第 3 次点火后，取第 2 个可用远地点作为第 4 次点火；
第 4 次后，第 5 次为 A→P 半圈转移近地点点火。
```

如果需要兼容旧版报告，也可配置：

```text
q = (3, 2, 3)
```

但本版建议先固定 q，而不是每次重新枚举，以减少变量维度和结果跳变。

## 4.3 第四、第五次固定半长轴控制

第 4 次脉冲后半长轴固定：

```text
a4_plus_fixed = 47271.168509 km
```

第 5 次脉冲后半长轴固定：

```text
a5_plus_fixed = 42164.2 km
```

如果工程输入给出的是“半长轴控制增量”，则使用：

```text
a4_plus = a4_minus + Δa4_fixed
a5_plus = a5_minus + Δa5_fixed
```

若工程输入给出的是“脉冲后半长轴目标”，则直接使用固定值。

# 5. 前三次机动量均匀分配原则

## 5.1 总前段机动量

定义前三次机动量合计：

```text
S123 = Δv1 + Δv2 + Δv3
```

推荐默认：

```text
S123_ref = 914 m/s
```

该值来自当前任务中“前三次重新分配后仍可满足固定尾段”的经验平台区。也可以按任务重新估算：

```text
S123_ref = S_total_est - Δv4_est - Δv5_est
```

其中：

```text
S_total_est = 五次脉冲总速度增量估计
Δv4_est, Δv5_est = 固定尾段按默认方向角估算得到的速度增量
```

## 5.2 完全均分初值

完全均分为：

```text
Δv_mean = S123_ref / 3
```

当：

```text
S123_ref = 914 m/s
```

则：

```text
Δv_mean ≈ 304.7 m/s
```

但考虑第一次远地点机动对降倾角和后续回归有较强影响，建议第一次略高于平均值。

## 5.3 推荐均匀分配模板

### 模板 A：图中第一次机动量保持型

适合与图中第一次变轨量一致，有限推力裕度较大。

```text
Δv1_ref = 325.4 m/s
Δv2_ref = 276.6 m/s
Δv3_ref = 312.0 m/s
S123 ≈ 914.0 m/s
```

特点：

```text
1. 第一次点火时间裕度好；
2. 第二次承担一定补偿；
3. 三次 Δv 差异不大；
4. 适合作为有限推力序贯优化初值。
```

### 模板 B：均衡中值型

适合追求更平滑的 Δv 分布。

```text
Δv1_ref = 330 m/s
Δv2_ref = 290 m/s
Δv3_ref = 294 m/s
S123 ≈ 914 m/s
```

特点：

```text
1. 三次点火量最均衡；
2. 每次点火均远离 90 min 上限；
3. 需要通过方向角和尾段校正保证终端约束。
```

### 模板 C：第一稍强型

适合希望第一次承担更多降倾角与升轨任务，但避免接近 90 min。

```text
Δv1_ref = 350 m/s
Δv2_ref = 235 m/s
Δv3_ref = 329 m/s
S123 ≈ 914 m/s
```

特点：

```text
1. 脉冲层面推进剂通常略优；
2. 第一次点火时长仍有裕度；
3. 第二次较小，第三次补偿较明显；
4. 仍属于可接受的“适度均匀”分配。
```

## 5.4 推荐默认方案

本版算法默认采用：

```text
Δv1_ref = 325.4 m/s
Δv2_ref = 276.6 m/s
Δv3_ref = 312.0 m/s
```

原因：

```text
1. 已接近图中第一次变轨量；
2. 最大点火时长裕度较好；
3. 三次分配相对平滑；
4. 推进剂损失相对最优方案可忽略；
5. 更适合后续有限推力展开。
```

# 6. 机动量约束

虽然不再以推进剂为优化目标，但仍要限制每次点火机动量，避免不可实现解。

## 6.1 第一次 Δv 约束

若点火时间包含沉底时间，则：

```text
τset + τmain ≤ 90 min
```

推荐第一次机动量范围：

```text
300 m/s ≤ Δv1 ≤ 416 m/s
```

工程建议范围：

```text
325 m/s ≤ Δv1 ≤ 360 m/s
```

## 6.2 第二、第三次 Δv 约束

为保证均匀分配与有限推力裕度：

```text
220 m/s ≤ Δv2 ≤ 340 m/s
250 m/s ≤ Δv3 ≤ 360 m/s
```

若采用模板 A，可进一步收窄为：

```text
250 m/s ≤ Δv2 ≤ 310 m/s
290 m/s ≤ Δv3 ≤ 340 m/s
```

## 6.3 均匀性约束

定义前三次平均机动量：

```text
Δv_bar = (Δv1 + Δv2 + Δv3) / 3
```

要求：

```text
|Δvk - Δv_bar| ≤ Δv_spread_max
```

推荐：

```text
Δv_spread_max = 70 m/s
```

若想更均匀：

```text
Δv_spread_max = 50 m/s
```

模板 A 中：

```text
Δv = [325.4, 276.6, 312.0] m/s
Δv_bar ≈ 304.7 m/s
最大偏差 ≈ 28.1 m/s
```

满足较严格均匀性要求。

# 7. 点火方向角处理

## 7.1 方向定义

第 k 次点火方向角 αk 定义在当地水平面内：

```text
u_k = cos(αk) Ehat + sin(αk) Shat
```

其中：

```text
Ehat = 当地正东方向
Shat = 当地正南方向
```

α 为正表示向南分量，用于降低倾角。

## 7.2 前三次方向角初值

推荐初值：

```text
α1 = 15 deg
α2 = 10 deg
α3 = 9 deg
```

推荐搜索范围：

```text
α1 ∈ [8 deg, 22 deg]
α2 ∈ [4 deg, 18 deg]
α3 ∈ [4 deg, 16 deg]
```

## 7.3 第四、第五次方向角

第 4、5 次半长轴控制固定，但方向角仍可用于调节终端倾角、偏心率和标称经度。

推荐：

```text
α4 初值 = 11.37 deg
α5 初值 = -176.29 deg
```

可选策略：

```text
策略 1：固定 α4、α5，只做终端校核；
策略 2：允许 α4、α5 做二维小范围校正；
策略 3：允许 α3、α4、α5 联合小范围校正。
```

本版默认采用策略 2：

```text
α4 ∈ [6 deg, 16 deg]
α5 ∈ [-180 deg, -165 deg]
```

# 8. 由固定 Δv 推导脉冲后状态

本版前三次不使用“目标半长轴反求 Δv”为主流程，而是直接使用固定或半固定 Δv。

在第 k 次点火点：

```text
v_k^+ = v_k^- + Δv_k * u_k
```

其中：

```text
u_k = cos(αk) Ehat + sin(αk) Shat
```

然后计算脉冲后轨道：

```text
a_k^+ = - μ / (2ε)
ε = |v_k^+|^2 / 2 - μ / r_k
```

```text
e_k^+ = || (v_k^+ × h_k^+) / μ - r_k / |r_k| ||
```

```text
i_k^+ = acos(h_z / |h|)
```

# 9. 第四、第五次固定半长轴脉冲

第 4、5 次仍采用“给定目标半长轴，反求 Δv”的方法。

## 9.1 反求公式

给定点火前状态：

```text
r, v
```

给定方向：

```text
u = cos(α)Ehat + sin(α)Shat
```

给定目标半长轴：

```text
a_plus_target
```

要求：

```text
v_plus = v + Δv u
```

满足：

```text
|v + Δv u|² / 2 - μ/r = -μ/(2a_plus_target)
```

展开为：

```text
Δv² + 2(v·u)Δv + [|v|² - 2μ/r + μ/a_plus_target] = 0
```

令：

```text
B = v·u
C = |v|² - 2μ/r + μ/a_plus_target
D = B² - C
```

则：

```text
Δv = -B ± sqrt(D)
```

选择：

```text
Δv > 0 且最小的根
```

若：

```text
D < 0
```

说明该方向无法达到目标半长轴，应判为不可行。

# 10. 目标函数设计

## 10.1 不再使用推进剂优化

本版不采用：

```text
min total_propellant
```

也不采用：

```text
min total_delta_v
```

推进剂只作为输出指标，不作为主优化目标。

## 10.2 新目标函数

目标函数改为：

```text
min J = P_terminal + P_uniform + P_duration + P_longitude + P_tail
```

其中：

```text
P_terminal  = 终端轨道误差罚函数
P_uniform   = 前三次 Δv 均匀性罚函数
P_duration  = 点火时长超限罚函数
P_longitude = 点火经度窗口罚函数
P_tail      = 第4/5次固定尾段可行性罚函数
```

## 10.3 终端误差罚函数

```text
P_terminal =
    Wa   * ((a_f - a_target) / a_scale)^2
  + We   * ((e_f - e_target) / e_scale)^2
  + Wi   * ((i_f - i_target) / i_scale)^2
  + Wlon * (wrap180(λ_f - λ_target) / λ_scale)^2
```

推荐尺度：

```text
a_scale   = 1 km
e_scale   = 1e-4
i_scale   = 0.01 deg
lon_scale = 0.05 deg
```

## 10.4 均匀性罚函数

```text
Δv_bar = (Δv1 + Δv2 + Δv3) / 3
```

```text
P_uniform =
    Wu * [ ((Δv1 - Δv_bar) / σdv)^2
         + ((Δv2 - Δv_bar) / σdv)^2
         + ((Δv3 - Δv_bar) / σdv)^2 ]
```

推荐：

```text
σdv = 50 m/s
```

若完全采用固定模板，则 `P_uniform` 可不参与优化，只用于结果检查。

## 10.5 时长罚函数

每次脉冲速度增量转换为主机时长估算：

```text
m_set = mdot_set * τset
m_after_set = m_before - m_set
Δv_set = Isp_set * g0 * ln(m_before / m_after_set)
Δv_main = max(0, Δv_impulse - Δv_set)
τmain = m_after_set / mdot_main_eff * [1 - exp(-Δv_main / c_main_eff)]
```

若点火时间包含沉底：

```text
τtotal = τset + τmain
```

约束：

```text
τtotal ≤ 90 min
```

罚函数：

```text
P_duration = Wt * Σ max(0, τtotal,k - 90min)^2
```

# 11. 优化变量

## 11.1 固定模板模式

最简模式：

```text
变量 y = [α1, α2, α3, α4, α5]
```

前三次 Δv 固定为模板值：

```text
Δv1 = 325.4 m/s
Δv2 = 276.6 m/s
Δv3 = 312.0 m/s
```

只优化方向角。

## 11.2 半固定均匀模式

推荐模式：

```text
变量 y = [δ1, δ2, α1, α2, α3, α4, α5]
```

其中：

```text
Δv1 = Δv1_ref + δ1
Δv2 = Δv2_ref + δ2
Δv3 = S123_ref - Δv1 - Δv2
```

并约束：

```text
|δ1| ≤ 25 m/s
|δ2| ≤ 25 m/s
```

这样既能保持前三次近似均匀，又允许小范围修正终端误差。

## 11.3 低维校正模式

若需要更高终端精度，可使用：

```text
变量 y = [Δv1, Δv2, α1, α2, α3, α4, α5]
```

其中：

```text
Δv3 = S123_ref - Δv1 - Δv2
```

约束：

```text
300 ≤ Δv1 ≤ 360
250 ≤ Δv2 ≤ 310
290 ≤ Δv3 ≤ 340
```

该模式仍不是推进剂优化，因为 `S123_ref` 固定，且目标函数不含推进剂项。

# 12. 推荐默认算法流程

## 12.1 输入

```text
X0, t0, m0
目标轨道：af, ef, if, lonf
发动机参数
沉底参数
q = (3, 3, 2)
a4_plus_fixed
a5_plus_fixed
前三次 Δv 模板
经度窗口
点火时长上限
```

## 12.2 主流程

```text
Step 1：初始化
    设置 q12=3, q23=3, q34=2
    设置 Δv1_ref=325.4 m/s
    设置 Δv2_ref=276.6 m/s
    设置 Δv3_ref=312.0 m/s
    设置 α 初值

Step 2：传播到第 1 次点火点
    从初始状态传播到第 2 圈远地点
    检查点火点经度是否在窗口内

Step 3：执行第 1 次脉冲
    使用 Δv1_ref 和 α1
    更新速度与质量
    计算后 a/e/i

Step 4：按 q12=3 寻找第 2 次远地点
    传播到第 3 个可用远地点
    检查经度窗口

Step 5：执行第 2 次脉冲
    使用 Δv2_ref 和 α2
    更新速度与质量

Step 6：按 q23=3 寻找第 3 次远地点
    传播到第 3 个可用远地点
    检查经度窗口

Step 7：执行第 3 次脉冲
    使用 Δv3_ref 和 α3
    更新速度与质量

Step 8：按 q34=2 寻找第 4 次远地点
    传播到第 2 个可用远地点
    检查经度窗口

Step 9：执行第 4 次固定半长轴脉冲
    给定 a4_plus_fixed
    根据 α4 反求 Δv4
    更新速度与质量

Step 10：传播到下一近地点
    检查第 5 次点火经度窗口

Step 11：执行第 5 次固定半长轴脉冲
    给定 a5_plus_fixed
    根据 α5 反求 Δv5
    更新速度与质量

Step 12：终端校核
    计算 af, ef, if, λf
    计算终端误差
    检查每次点火时长
    输出点火表
```

# 13. 方向角校正流程

## 13.1 固定 Δv，仅优化方向角

定义：

```text
y = [α1, α2, α3, α4, α5]
```

目标：

```text
min P_terminal + P_longitude + P_duration
```

约束：

```text
α1 ∈ [8, 22] deg
α2 ∈ [4, 18] deg
α3 ∈ [4, 16] deg
α4 ∈ [6, 16] deg
α5 ∈ [-180, -165] deg
```

推荐优化器：

```text
Powell / Nelder-Mead / COBYLA / SLSQP
```

## 13.2 小范围 Δv + 方向角联合校正

定义：

```text
y = [δ1, δ2, α1, α2, α3, α4, α5]
```

其中：

```text
Δv1 = 325.4 + δ1
Δv2 = 276.6 + δ2
Δv3 = 914.0 - Δv1 - Δv2
```

约束：

```text
-25 ≤ δ1 ≤ 25
-25 ≤ δ2 ≤ 25
290 ≤ Δv3 ≤ 340
```

目标：

```text
min P_terminal + P_uniform + P_duration + P_longitude
```

注意：目标函数中仍不加入推进剂项。

# 14. 经度窗口约束

## 14.1 脉冲点经度窗口

基础窗口：

```text
40 degE ≤ λ ≤ 180 degE
```

脉冲规划建议使用收缩窗口：

```text
45 degE ≤ λ ≤ 175 degE
```

如果某个脉冲点贴近边界，则应优先调整 q 或方向角，不建议仅通过增加 Δv 强行修正。

## 14.2 有限推力展开裕度

由于脉冲点后续会转换为有限推力弧段，建议在脉冲阶段预留经度裕度：

```text
λ_margin ≥ 5 deg
```

即：

```text
45 degE ≤ λ_impulse ≤ 175 degE
```

若点火弧段较长，则应进一步收缩为：

```text
50 degE ≤ λ_impulse ≤ 170 degE
```


这样转入有限推力积分优化后仍有时间调整空间。

# 16. 推进剂计算

推进剂不参与主优化，但必须输出用于工程评估。

## 16.1 单次推进剂

沉底消耗：

```text
mp_set = mdot_set * τset
```

沉底贡献速度增量：

```text
Δv_set = Isp_set*g0*ln(m_before / (m_before - mp_set))
```

主机需要承担：

```text
Δv_main = max(0, Δv_impulse - Δv_set)
```

主机消耗：

```text
mp_main = m_after_set * [1 - exp(-Δv_main / c_main_eff)]
```

单次总消耗：

```text
mp_k = mp_set + mp_main
```

质量更新：

```text
m_after = m_before - mp_k
```

# 18. 完整伪代码

```text
function simplified_impulse_planner_v4(config):

    load constants and initial state

    set q = (3, 3, 2)

    set fixed tail:
        a4_plus_fixed = 47271.168509 km
        a5_plus_fixed = 42164.2 km

    set front Δv template:
        dv1 = 325.4 m/s
        dv2 = 276.6 m/s
        dv3 = 312.0 m/s

    set alpha initial:
        alpha1 = 15 deg
        alpha2 = 10 deg
        alpha3 = 9 deg
        alpha4 = 11.37 deg
        alpha5 = -176.29 deg

    define decision variable:
        mode A:
            y = [alpha1, alpha2, alpha3, alpha4, alpha5]
        mode B:
            y = [delta1, delta2, alpha1, alpha2, alpha3, alpha4, alpha5]

    define evaluate(y):

        construct dv1, dv2, dv3
        check front Δv uniformity
        check estimated burn duration

        X = X0
        m = m0
        t = t0

        # burn 1
        T1 = find_initial_valid_apogee(X, t)
        check longitude(T1)
        X = propagate_to(T1)
        X, m = apply_impulse(X, m, dv1, alpha1)

        # burn 2
        T2 = find_qth_valid_apogee(X, T1, q12)
        check longitude(T2)
        X = propagate_to(T2)
        X, m = apply_impulse(X, m, dv2, alpha2)

        # burn 3
        T3 = find_qth_valid_apogee(X, T2, q23)
        check longitude(T3)
        X = propagate_to(T3)
        X, m = apply_impulse(X, m, dv3, alpha3)

        # burn 4 fixed a+
        T4 = find_qth_valid_apogee(X, T3, q34)
        check longitude(T4)
        X = propagate_to(T4)
        dv4 = solve_dv_for_target_a(X, alpha4, a4_plus_fixed)
        X, m = apply_impulse(X, m, dv4, alpha4)

        # burn 5 fixed a+
        T5 = find_next_perigee(X, T4)
        check longitude(T5)
        X = propagate_to(T5)
        dv5 = solve_dv_for_target_a(X, alpha5, a5_plus_fixed)
        X, m = apply_impulse(X, m, dv5, alpha5)

        terminal = compute_orbit_and_nominal_longitude(X, T5)

        P_terminal = terminal_penalty(terminal)
        P_uniform = uniformity_penalty(dv1, dv2, dv3)
        P_duration = duration_penalty(all_burns)
        P_longitude = longitude_penalty(all_burns)

        J = P_terminal + P_uniform + P_duration + P_longitude

        return J, result_table

    run optimizer:
        if fixed template mode:
            optimize alpha only
        else:
            optimize small dv correction + alpha

    return best result
```

# 19. 推荐实施步骤

## 19.1 第一步：固定模板快速计算

使用：

```text
Δv1 = 325.4 m/s
Δv2 = 276.6 m/s

```

只优化：

```text

```

输出终端误差。

## 19.2 第二步：小范围修正

若终端误差未满足：

```text
|da| ≤ 1 km
e ≤ 1e-4
|di| ≤ 0.01 deg
|dlon| ≤ 0.05 deg
```

则释放：

```text
δ1, δ2
```

允许：

```text
Δv1 = 325.4 ± 25 m/s
Δv2 = 276.6 ± 25 m/s
Δv3 = 914.0 - Δv1 - Δv2
```

## 19.3 第三步：转入有限推力初值

脉冲转有限推力时：

```text
脉冲时刻 = 主发动机点火弧段中心
主机开始 = 脉冲时刻 - τmain/2
主机结束 = 脉冲时刻 + τmain/2
沉底开始 = 主机开始 - 240 s
```

不要把脉冲时刻直接当作沉底开始时刻。

# 22. 结论

本版简化脉冲规划算法将问题从“推进剂最优”改为“工程稳健初值生成”。

核心原则是：

```text
第 4、5 次固定半长轴控制；
第 1、2、3 次不再追求推进剂微小最优；
前三次 Δv 采用适当均匀分配；
只通过方向角和小范围 Δv 修正满足终端约束；
优先保证最长点火时间、经度窗口和有限推力展开裕度。
```

推荐默认前三次分配为：

```text
Δv1 = 325.4 m/s
Δv2 = 276.6 m/s
Δv3 = 312.0 m/s
```

若需要更均衡，也可使用：

```text
Δv1 = 330 m/s
Δv2 = 290 m/s
Δv3 = 294 m/s
```

实际工程流程建议为：

```text
固定模板快速计算
    ↓
方向角校正
    ↓
必要时小范围 Δv 修正
    ↓
固定第 4/5 次半长轴尾段校核
    ↓
输出有限推力初值
```
