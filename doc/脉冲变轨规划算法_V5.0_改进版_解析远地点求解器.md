# 脉冲变轨规划算法 V5.0 详细设计文档  
## 相位约束驱动式序贯脉冲规划

版本：V5.0  
文档类型：工程算法详细设计  
适用对象：从高椭圆转移轨道、超同步转移轨道或标准同步转移轨道，经多次脉冲机动生成有限推力优化初值。  
核心目标：在满足测控经度窗口、点火时长、终端轨道与经度相位约束的前提下，生成结构清晰、动力学一致、可用于有限推力序贯优化的脉冲初始解。

---

# 0. 总体思想

V5.0 脉冲变轨规划算法采用：

```text
q 序列 + 控后半长轴 / 控近地点高度 + 序贯 alpha 优化
```

的相位约束驱动策略。

算法不再把主要问题描述为“把总 Δv 均匀分成若干份”，而是将每次远地点点火后的轨道周期、下一拱点出现时刻、星下点经度、终端标称经度等全部纳入规划。

核心逻辑为：

```text
1. 先寻找满足测控经度窗口的第一个远地点；
2. 在该远地点计算一次远地点脉冲完成近地点抬升与倾角修正所需的参考 Δv；
3. 用该参考 Δv 推荐远地点点火次数；
4. 用 q 序列控制相邻点火之间的飞行圈次；
5. 用每次点火后的目标半长轴 / 近地点高度控制下一次点火点经度；
6. 逐次优化每次远地点点火的 alpha；
7. 最后一次远地点点火完成近地点同步高度和目标倾角；
8. 若为超同步转移轨道，再在近地点执行一次或多次面内降远地点点火；
9. 最后对全部脉冲参数进行小范围整体精修；
10. 输出可转换为有限推力初值的脉冲表。
```

---

# 1. 适用范围与轨道类型

## 1.1 目标轨道

默认目标为近圆同步轨道：

```text
a_target = r_sync ≈ 42164.2 km
e_target = 0
i_target = 用户指定
lambda_target = 用户指定标称经度
```

其中：

```text
r_sync = (mu / omega_e^2)^(1/3)
```

工程上也可以直接配置：

```text
r_sync = 42164.2 km
```

若任务目标为倾斜同步轨道，则：

```text
i_target > 0
```

此时“定点经度”应理解为标称平均经度，而不是瞬时星下点经度。

---

## 1.2 超同步转移轨道

若初始远地点半径满足：

```text
r_a0 > r_sync + supersync_margin_km
```

则判定为超同步转移轨道。

默认：

```text
supersync_margin_km = 500 km
```

结构为：

```text
A, A, ..., A, P
```

其中：

```text
A = 远地点点火
P = 近地点点火
```

若最后一次近地点点火超出单次点火能力，则自动转入多近地点尾段结构：

```text
A, A, ..., A, P, P, ..., P
```

---

## 1.3 严格标准转移轨道

若初始远地点半径满足：

```text
|r_a0 - r_sync| <= strict_standard_apogee_tolerance_km
```

则可判定为严格标准转移轨道。

推荐：

```text
strict_standard_apogee_tolerance_km = 5 km
```

结构为：

```text
A, A, ..., A
```

因为最后一次远地点点火位置半径已经足够接近同步半径，具备直接圆化为同步轨道的几何可达性。

---

## 1.4 近标准转移轨道

若：

```text
strict_standard_apogee_tolerance_km < |r_a0 - r_sync| <= standard_apogee_margin_km
```

则为近标准转移轨道。

默认：

```text
standard_apogee_margin_km = 500 km
```

注意：近标准转移轨道不一定能在最后一次远地点点火后严格满足：

```text
a = r_sync
e = 0
```

原因是脉冲点火不会改变当前位置半径。若最后一次远地点点火时：

```text
r_burn != r_sync
```

则无法通过一次脉冲立刻得到半径为 `r_sync` 的圆轨道。

近标准转移轨道的处理方式有三种：

```text
1. 放宽终端偏心率，允许小偏心率同步轨道；
2. 增加终端修正脉冲；
3. 转入通用转移轨道模式。
```

---

## 1.5 通用转移轨道

不满足以上分类时，进入通用转移轨道模式。通用模式不强制只在远地点或近地点点火，可以扩展为：

```text
A/P 自动拱点序列
```

本文档重点描述超同步转移轨道与严格标准转移轨道。

# 3. 输入参数

## 3.1 初始轨道

支持两类输入：

```text
1. Keplerian:
   a, e, i, Ω, ω, M 或 f

2. Cartesian:
   r_ECI, v_ECI
```

若输入为开普勒根数，先转换为 ECI 状态。

若给定升交点地理经度而不是惯性升交点赤经，则需根据初始历元地球自转角换算：

```text
Ω_ECI = Ω_geo + θG(t0)
```

角度需统一 wrap 到 `[0, 360)` 或 `[-180, 180)`。

---

## 3.2 目标轨道

| 参数 | 符号 | 默认 / 说明 |
|---|---:|---|
| 目标半长轴 | a_target | 42164.2 km |
| 目标偏心率 | e_target | 0 |
| 目标倾角 | i_target | 用户指定 |
| 目标标称经度 | λ_target | 用户指定 |
| 终端经度类型 | longitude_target_type | mean_longitude |
| 终端漂移率要求 | drift_rate_target | 0 deg/day |

目标近地点半径：

```text
r_p,target = a_target (1 - e_target)
```

目标远地点半径：

```text
r_a,target = a_target (1 + e_target)
```

对于近圆同步轨道：

```text
r_p,target = r_a,target = a_target = r_sync
```

---

## 3.3 发动机与质量参数

| 参数 | 符号 | 默认值 | 说明 |
|---|---:|---:|---|
| 初始质量 | m0 | 用户指定 | kg |
| 主发动机推力 | F_main | 490 N | 可配置 |
| 主发动机比冲 | Isp_main | 314.1 s | 可配置 |
| 姿控效率修正 | η_ac | 0.0173 | 可配置 |
| 沉底推力 | F_set | 20 N | 可配置 |
| 沉底比冲 | Isp_set | 290 s | 可配置 |
| 沉底时长 | τ_set | 240 s | 可配置 |

姿控效率修正用于等效降低主发动机比冲：

```text
Isp_main_eff = Isp_main / (1 + η_ac)
```

主发动机等效喷速：

```text
c_main_eff = Isp_main_eff * g0
```

主发动机等效质量流率：

```text
mdot_main_eff = F_main / c_main_eff
```

沉底质量流率：

```text
mdot_set = F_set / (Isp_set * g0)
```

姿控效率 `η_ac` 应由任务历史统计或推进剂预算确定。若无姿控效率修正，令：

```text
η_ac = 0
```

## 3.4 经度窗口

| 参数 | 默认 |
|---|---:|
| 硬测控窗口 | [40, 180] degE |
| 脉冲规划窗口 | [45, 175] degE |
| 有限推力预留窗口 | [50, 170] degE |

脉冲规划阶段推荐使用收缩窗口：

```text
lon_window_planning = [45, 175] degE
```

以便后续有限推力弧段展开时仍能满足硬窗口。

## 3.5 点火次数与运行模式

| 参数 | 默认 | 说明 |
|---|---:|---|
| auto_recommend_count | true | 自动推荐次数 |
| maneuver_count_user | 0 | 用户指定总次数，0 表示未指定 |
| max_burn_count | 12 | 最大总次数 |
| max_q_between_burns | 3 | 相邻点火最大飞行圈次 |
| runtime_mode | normal | fast / normal / high_quality |

运行模式：

```text
fast:
    默认 q=3，仅局部修复；
    alpha 粗步长 2 deg + 精修；
    不做大范围 q 搜索。

normal:
    q 序列 beam search；
    alpha 粗步长 1 deg + 精修；

high_quality:
    更宽 beam；

    更严格终端容差；
    适合离线生成最终初值。
```

# 4. 动力学模型

## 4.1 状态变量

```text
X = [r, v, m]

r = [x, y, z]^T       km
v = [vx, vy, vz]^T    km/s
m                       kg
```

---

## 4.2 二体动力学

非点火段基础模型：

```text
r_dot = v
v_dot = - μ r / |r|^3
m_dot = 0
```

---

## 4.3 J2 摄动

```text
r = sqrt(x² + y² + z²)
```

J2 加速度：

```text
a_J2,x = J2 * μ * x * Re² / r⁵ * [-1.5 + 7.5 (z/r)²]

a_J2,y = J2 * μ * y * Re² / r⁵ * [-1.5 + 7.5 (z/r)²]

a_J2,z = J2 * μ * z * Re² / r⁵ * [-4.5 + 7.5 (z/r)²]
```

非点火段：

```text
v_dot = - μ r / |r|^3 + a_J2
```

脉冲规划阶段默认使用：

```text
二体 + J2
```

忽略项：

```text
1. 日月第三体引力；
2. 太阳光压；
3. 高阶非球形重力场；
4. 潮汐与相对论项。
```

说明：V5.0 的定位是生成有限推力优化初值。若任务弧长较长、远地点显著高于同步高度、或终端经度精度要求很高，最终有限推力精修阶段应加入更高保真摄动模型。

---

## 4.4 有限推力动力学

沉底段：

```text
v_dot = - μ r / |r|^3 + a_J2 + F_set / m / 1000 * u_hat
m_dot = - F_set / (Isp_set * g0)
```

主发动机段：

```text
v_dot = - μ r / |r|^3 + a_J2 + F_main / m / 1000 * u_hat
m_dot = - F_main / (Isp_main_eff * g0)
```

其中 `u_hat` 位于当地水平面内。

---

# 5. 当地水平面与 alpha 定义

## 5.1 基本定义

```text
Rhat = r / |r|
Khat = [0, 0, 1]^T
```

通常情况下：

```text
Ehat = normalize(Khat × Rhat)
Nhat = normalize(Khat - (Khat · Rhat) Rhat)
Shat = -Nhat
```

其中：

```text
Ehat = 当地正东方向
Shat = 当地正南方向
```

alpha 定义：

```text
u_hat = cos(alpha) Ehat + sin(alpha) Shat
```

alpha 为正表示推力方向含当地正南分量。

---

## 5.2 奇点保护

当：

```text
|Khat × Rhat| < eps_horizontal
```

说明位置接近地理极区，当地正东方向定义奇异。此时采用备用构造：

```text
Vh = v - (v · Rhat) Rhat

if |Vh| > eps_horizontal:
    Ehat = normalize(Vh)
else:
    Ehat = any_unit_vector_perpendicular_to(Rhat)

Shat = normalize(Ehat × Rhat)
```

说明：极区严格地理东向本身存在定义奇异性。备用定义以当地水平速度投影作为切向参考方向，目的是保证动力学积分和优化过程不中断。

---

## 5.3 倾角修正方向规则

不能全局硬编码：

```text
alpha >= 0
```

作为降倾角条件。因为 alpha 的物理效果取决于当前轨道面、点火点纬度、速度方向和角动量方向。

V5.0 使用如下规则：

```text
若本次远地点点火目标包含降倾角，则候选 alpha 必须使倾角误差不显著增加。
```

即：

```text
|i_plus - i_target| <= |i_before - i_target| + i_relax_tol
```

默认：

```text
i_relax_tol = 0.02 deg
```

对于特定任务，可设置 alpha 符号软引导：

```text
alpha_sign_hint = positive / negative / none
```

但该引导不能作为通用硬约束。

# 6. 经度定义

## 6.1 星下点经度

将 ECI 位置转换为 ECEF：

```text
r_ECEF = R3[-θG(t)] r_ECI
```

星下点经度：

```text
lambda_ss = atan2(y_ECEF, x_ECEF)
```

统一转换为：

```text
lambda_ss ∈ [0, 360) degE
```

点火窗口约束使用星下点经度：

```text
lambda_ss ∈ lon_window
```

---

## 6.2 地球自转角

工程实现可采用线性模型：

```text
θG(t) = θG(t0) + omega_e * (t - t0)
```

若任务需要更高精度，可使用 GMST / GAST。文档默认采用 GMST 近似，最终高精度仿真可由任务平台统一处理岁差、章动与极移。

---

## 6.3 终端标称经度

终端同步轨道的“定点经度”不能简单等同于某一瞬时星下点经度。V5.0 默认使用平均经度：

```text
lambda_mean = Ω + ω + M - θG(t)
```

其中：

```text
Ω = 升交点赤经
ω = 近地点幅角
M = 平近点角
```

终端经度约束：

```text
wrap180(lambda_mean - lambda_target) = 0
```

---

## 6.4 近圆轨道下的非奇异处理

当：

```text
e < e_singular_threshold
```

经典根数中的 `ω` 和 `M` 单独不稳定。此时应使用非奇异平均经度，例如 equinoctial 元素中的 mean longitude。

工程上也可以采用数值平均法：

```text
1. 从终端状态向前传播若干采样点；
2. 计算每个采样点的瞬时平均经度；
3. 对短弧段平均，得到 terminal lambda_nom；
4. 用该值约束终端标称经度。
```

推荐默认：

```text
e_singular_threshold = 1e-4
```

---

## 6.5 终端经度漂移率

同步终端不仅要求标称经度接近目标，还要求平均经度漂移率接近 0。

有限差分计算：

```text
dot_lambda_mean ≈ wrap180(lambda_mean(t + Δt) - lambda_mean(t)) / Δt
```

默认：

```text
Δt = 3600 s
```

漂移率约束：

```text
|dot_lambda_mean| <= drift_rate_tol
```

默认：

```text
drift_rate_tol = 0.01 deg/day
```

---

# 7. 轨道根数与拱点事件

## 7.1 轨道能量

```text
epsilon = |v|² / 2 - μ / |r|
```

半长轴：

```text
a = - μ / (2 epsilon)
```

---

## 7.2 角动量与倾角

```text
h = r × v
i = acos(h_z / |h|)
```

---

## 7.3 偏心率向量

```text
e_vec = (v × h) / μ - r / |r|
e = |e_vec|
```

近地点半径：

```text
r_p = a (1 - e)
```

远地点半径：

```text
r_a = a (1 + e)
```

---

## 7.4 拱点事件

径向速度：

```text
v_r = r · v / |r|
```

远地点事件：

```text
v_r 从正变负
```

近地点事件：

```text
v_r 从负变正
```

事件搜索应使用带根定位的传播器，而不是固定步长近似取点。

# 8. 脉冲模型

## 8.1 脉冲速度更新

第 k 次脉冲：

```text
v_k^+ = v_k^- + Δv_k * u_hat_k
```

其中：

```text
u_hat_k = cos(alpha_k) Ehat_k + sin(alpha_k) Shat_k
```

---

## 8.2 质量更新

脉冲规划阶段可使用火箭方程估计质量消耗。

每次脉冲先扣除沉底推进剂：

```text
m_after_set = m_before - mdot_set * τ_set
```

沉底等效速度增量：

```text
dv_set = Isp_set * g0 * ln(m_before / m_after_set)
```

主发动机承担：

```text
dv_main = max(0, dv_impulse - dv_set)
```

主发动机推进剂消耗：

```text
m_after_main = m_after_set * exp(-dv_main / c_main_eff)
```

总消耗：

```text
mp = m_before - m_after_main
```

注意：脉冲规划中的质量消耗用于点火时长与推进剂估算。真正有限推力优化应按连续推力动力学重新积分。

# 9. 固定脉冲后半长轴反求 Δv

给定点火前状态 `r, v`，方向 `u`，目标脉冲后半长轴 `a_plus_target`。

要求：

```text
v_plus = v + Δv u
```

满足：

```text
|v_plus|² / 2 - μ / |r| = - μ / (2 a_plus_target)
```

展开：

```text
Δv² + 2B Δv + C = 0
```

其中：

```text
B = v · u

C = |v|² - 2μ/|r| + μ/a_plus_target
```

判别式：

```text
D = B² - C
```

根：

```text
Δv = -B ± sqrt(D)
```

取根规则：

```text
1. 若 D < 0，则该 alpha 和目标 a_plus 不可行；
2. 保留所有 Δv > 0 的根；
3. 优先选择最小正根；
4. 对候选根进行几何检查；
5. 若最小正根不满足几何目标，则判定当前候选不可行，不直接选择大根。
```

两个正根的解释：

```text
较小正根：
    沿给定方向以最小速度增量达到目标能量，通常为工程合理解。

较大正根：
    速度矢量沿 u 方向越过二次函数最低点后再次达到同一能量；
    虽然能量仍满足目标半长轴，但可能导致速度方向分量反转、角动量或偏心率异常；
    一般不作为拱点变轨解。
```

# 10. 远地点一次完成近地点抬升与倾角修正解析求解器

## 10.1 问题定义

在远地点状态 `state_A = [r, v]`，求一次水平面内脉冲，使脉冲后满足：

```text
r_p^+ = r_p,target
i^+   = i_target
```

并使：

```text
Δv = |v^+ - v^-|
```

最小。

返回：

```text
dva_ref, alpha_ref
```

或用于最后一次远地点点火：

```text
dv_n, alpha_n
```

其中：

```text
alpha_ref / alpha_n 是脉冲增量方向角，不是点火后速度方向角。
```

该问题在以下条件下有解析解：

```text
1. 点火点为远地点或足够接近远地点，径向速度 v_r ≈ 0；
2. 脉冲方向限制在当地水平面内，无径向分量；
3. 点火后当前位置仍作为新轨道的远地点；
4. 仅约束目标近地点半径和目标倾角；
5. 不在该求解器内部约束 RAAN、近地点幅角、终端平均经度。
```

终端经度、RAAN 间接效果和点火点相位由后续 `q_seq + a_plus/rp_plus` 相位求解器处理。

---

## 10.2 远地点不是升交点时的几何处理

远地点一般不是升交点，也不一定是降交点。不能把远地点点火简化为“在节点处做倾角修正”。

本求解器不使用升交点假设，而是直接在点火点的当地水平坐标系中求过该空间点的目标倾角轨道面。

定义：

```text
Rhat = r / |r|
Khat = [0, 0, 1]^T
```

当地东、北、南方向：

```text
Ehat = normalize(Khat × Rhat)
Nhat = normalize(Khat - (Khat·Rhat)Rhat)
Shat = -Nhat
```

其中 `Ehat` 为当地正东，`Nhat` 为当地正北，`Shat` 为当地正南。

点火点地心纬度为：

```text
sin δ = Rhat_z
cos δ = sqrt(1 - Rhat_z²)
```

点火后速度方向写成：

```text
vhat^+ = cos β Ehat + sin β Shat
```

这里：

```text
β 是点火后速度方向角；
β 不是脉冲方向角 alpha。
```

点火后角动量方向：

```text
hhat^+ = Rhat × vhat^+
        = cos β Nhat + sin β Ehat
```

由于：

```text
Ehat_z = 0
Nhat_z = cos δ
```

所以：

```text
cos i^+ = hhat_z^+ = cos β cos δ
```

要求 `i^+ = i_target`，得到：

```text
cos β = cos(i_target) / cos δ
```

因此两个解析候选为：

```text
β_1 = + arccos[cos(i_target) / cos δ]
β_2 = - arccos[cos(i_target) / cos δ]
```

这两个候选不是“升交点 / 降交点”本身，而是过当前远地点位置的两条目标倾角轨道切向方向。只有当 `δ ≈ 0`，即点火点接近赤道交点时，它们才可近似理解为升支 / 降支方向。

选择哪一个候选，应由实际所需脉冲最小或与当前速度方向最接近决定，而不是预设远地点为升交点。

---

## 10.3 可达性条件

解析候选存在的必要条件为：

```text
|cos(i_target) / cos δ| <= 1
```

等价于：

```text
|cos(i_target)| <= cos δ
```

对于顺行低倾角任务，可近似理解为：

```text
i_target >= |δ|
```

物理意义：

```text
倾角为 i_target 的轨道无法经过地心纬度绝对值大于 i_target 的空间点。
```

因此，如果某个远地点纬度 `|δ_A|` 已经大于目标倾角 `i_target`，则不可能在该远地点一次点火后直接得到目标倾角轨道。此时应返回：

```text
INCLINATION_NOT_REACHABLE_AT_THIS_APOGEE_LATITUDE
```

而不是强行搜索 alpha。

处理策略：

```text
1. 更换后续远地点，即调整 q_seq；
2. 选择纬度更低的远地点作为终端远地点；
3. 增加远地点点火次数，使倾角逐步降低；
4. 放宽“最后一次远地点一次达到目标倾角”的要求，转入整体精修；
5. 若任务必须在当前远地点完成，则判定几何不可行。
```

---

## 10.4 由目标近地点半径确定点火后速度大小

在远地点点火后，当前位置半径作为新轨道远地点半径：

```text
r_a^+ = r_A = |r|
```

目标近地点半径：

```text
r_p^+ = r_p,target
```

脉冲后半长轴：

```text
a^+ = (r_A + r_p,target) / 2
```

由 vis-viva 方程：

```text
(v^+)² = μ (2/r_A - 1/a^+)
```

得到所需速度大小：

```text
v_req = sqrt[ μ (2/r_A - 1/a^+) ]
```

可行性要求：

```text
r_p,target < r_A
v_req² > 0
```

---

## 10.5 构造候选点火后速度

由第 10.2 节得到两个候选速度方向：

```text
β_1 = + arccos[cos(i_target) / cos δ]
β_2 = - arccos[cos(i_target) / cos δ]
```

构造候选点火后速度：

```text
v_plus,1 = v_req [cos β_1 Ehat + sin β_1 Shat]

v_plus,2 = v_req [cos β_2 Ehat + sin β_2 Shat]
```

对每个候选，计算脉冲增量：

```text
Δv_vec,j = v_plus,j - v^-
Δv_j     = |Δv_vec,j|
```

选择：

```text
j* = argmin Δv_j
```

返回：

```text
dv = Δv_j*
```

脉冲方向角为：

```text
alpha = atan2(Δv_vec,j* · Shat, Δv_vec,j* · Ehat)
```

注意：

```text
alpha ≠ β
```

`β` 是点火后速度方向角；`alpha` 是脉冲增量方向角，也是后续有限推力初值应使用的方向角。

---

## 10.6 候选分支选择规则

两个候选分支均满足目标近地点半径和目标倾角，但对应不同的局部飞行方向。

默认选择规则：

```text
1. 优先选择 Δv 最小的候选；
2. 若 Δv 差异小于 dv_branch_tie_tol，则选择与当前速度方向夹角更小的候选；
3. 若任务要求保持当前升降支方向，则增加 branch_policy 约束；
4. 若候选导致后续相位求解不可行，则可在 phase_score 中尝试另一分支。
```

建议配置：

```yaml
apogee_one_shot_solver:
  branch_policy: min_dv        # min_dv / closest_heading / preserve_branch / try_both
  dv_branch_tie_tol_mps: 1.0
```

对于相位敏感任务，推荐在 `phase_score` 中支持 `try_both`：

```text
同一个远地点 one-shot 剩余代价评估中，可把两个解析候选都作为分支，
由后续经度相位评分决定保留哪一个。
```

---

## 10.7 远地点事件误差处理

数值积分得到的远地点事件可能存在微小径向速度：

```text
v_r = v · Rhat ≠ 0
```

处理顺序：

```text
1. 优先使用事件定位器重新精修远地点，使 |v_r| <= radial_velocity_tol；
2. 若仍有极小残差，可将当前速度投影到当地水平面用于解析诊断；
3. 若 |v_r| 超过阈值，则返回 NOT_EXACT_APOGEE，不执行解析解。
```

默认阈值：

```text
radial_velocity_tol = 1.0e-8 km/s
```

---

## 10.8 极区奇点保护

当地正东方向定义：

```text
Ehat = normalize(Khat × Rhat)
```

当 `Rhat` 接近 `Khat` 时会出现奇点。虽然同步转移任务通常不会到达极区，但通用程序必须加保护。

若：

```text
|Khat × Rhat| < local_frame_eps
```

则返回：

```text
LOCAL_EAST_SINGULAR
```

或使用备用当地水平参考：

```text
That = normalize(v - (v·Rhat)Rhat)
Bhat = normalize(Rhat × That)
```

但注意备用参考下 `alpha` 不再是严格“东-南”角，应仅用于避免积分中断，不建议用于最终工程输出。

默认：

```text
local_frame_eps = 1.0e-10
```

---

## 10.9 解析求解器伪代码

```text
function analytic_solve_apogee_to_rp_i(state_A, rp_target, i_target, cfg):

    r = state_A.r
    v = state_A.v

    r_A = norm(r)
    Rhat = r / r_A

    v_radial = dot(v, Rhat)

    if abs(v_radial) > radial_velocity_tol:
        return infeasible(NOT_EXACT_APOGEE)

    if rp_target >= r_A:
        return infeasible(RP_TARGET_NOT_BELOW_APOGEE)

    Khat = [0, 0, 1]
    cross_KR = cross(Khat, Rhat)

    if norm(cross_KR) < local_frame_eps:
        return infeasible(LOCAL_EAST_SINGULAR)

    Ehat = normalize(cross_KR)
    Nhat = normalize(Khat - dot(Khat, Rhat) * Rhat)
    Shat = -Nhat

    cos_delta = sqrt(max(0, 1 - Rhat_z^2))

    cos_beta = cos(i_target) / cos_delta

    if abs(cos_beta) > 1 + inclination_reachability_eps:
        return infeasible(INCLINATION_NOT_REACHABLE_AT_THIS_APOGEE_LATITUDE)

    cos_beta = clamp(cos_beta, -1, 1)

    beta_abs = arccos(cos_beta)

    beta_candidates = [+beta_abs, -beta_abs]

    a_plus = 0.5 * (r_A + rp_target)
    v_req_sq = mu * (2/r_A - 1/a_plus)

    if v_req_sq <= 0:
        return infeasible(INVALID_VIS_VIVA)

    v_req = sqrt(v_req_sq)

    candidates = []

    for beta in beta_candidates:

        v_plus = v_req * (cos(beta) * Ehat + sin(beta) * Shat)

        dv_vec = v_plus - v
        dv = norm(dv_vec)

        dv_radial = dot(dv_vec, Rhat)

        if abs(dv_radial) > horizontal_impulse_tol:
            continue

        alpha = atan2(dot(dv_vec, Shat), dot(dv_vec, Ehat))

        heading_angle_error = angle_between(v_plus, v)

        candidates.append({
            dv, alpha, beta, a_plus, rp_target, v_req, heading_angle_error
        })

    if candidates is empty:
        return infeasible(NO_HORIZONTAL_IMPULSE_SOLUTION)

    selected = select_branch(candidates, branch_policy)

    return selected
```

输出字段：

```text
dv_mps
alpha_deg                 # 脉冲方向角
beta_velocity_deg         # 点火后速度方向角，仅用于诊断
a_plus_km
rp_plus_km
branch_id
status
```

## 10.10 与 V5.0 主流程的关系

该解析求解器替代原先的“外层 alpha 搜索 + 内层 Δv 求根”方案，用于两个位置：

```text
1. 第一个有效远地点的参考总 Δv：
   dva_ref, alpha_ref = analytic_solve_apogee_to_rp_i(A_1)

2. 最后一次远地点点火：
   dv_n, alpha_n = analytic_solve_apogee_to_rp_i(A_n)
```

在前段远地点点火中，若目标是固定 `a_plus` 而不是固定 `rp + i`，仍使用：

```text
solve_dv_for_post_a(state_Ak, alpha, a_plus_target)
```

并继续对 alpha 做一维优化。

因此，V5.0 中只有“参考 one-shot 解”和“最后远地点终端解”改为解析求解；前段固定 `a_plus` 的 alpha 优化仍保留。

A_1 = 第一个满足 lambda_ss(A_1) ∈ lon_window_planning 的远地点

## 11.2 算法

```text
function find_first_valid_apogee(X0, t0, cfg):

    X = X0
    t = t0

    while t - t0 <= max_search_days * 86400:

        X = event.X

```

# 12. 参考远地点总 Δv 与次数推荐

## 12.1 参考远地点总 Δv

在第一个有效远地点 `A_1`，调用：

```text
analytic_solve_apogee_to_rp_i(
    state_A1,
    r_p,target = r_sync,
    i_target
)
```

得到：

```text
dva_ref
alpha_ref
```

含义：

```text
dva_ref 是在第一个远地点一次性完成近地点抬升和倾角修正的最小速度增量。
```

它只用于：

```text
1. 推荐远地点点火次数；
2. 给第一段 alpha 优化提供初值；
3. 检查前 n 次远地点点火 Δv 总和是否异常。
```

不能把：

```text
dva_ref / n
```

直接作为每次点火量。

---

## 12.2 单次点火能力估计

主发动机最大点火时间：

```text
tau_main_max = max_total_burn_time_s - tau_set
```

沉底后质量：

```text
m_after_set = m - mdot_set * tau_set
```

主发动机最大消耗：

```text
mp_main_max = mdot_main_eff * tau_main_max
```

主发动机最大 Δv：

```text
dv_main_max = c_main_eff * ln(m_after_set / (m_after_set - mp_main_max))
```

沉底等效 Δv：

```text
dv_set = Isp_set * g0 * ln(m / m_after_set)
```

单次最大 Δv：

```text
dv_burn_max = dv_set + dv_main_max
```

设计单次 Δv：

```text
dv_burn_design = min(
    design_dv_per_burn,
    burn_utilization * dv_burn_max
)
```

---

## 12.3 远地点次数推荐

超同步转移轨道：

```text
n_A_raw = ceil(dva_ref / dv_burn_design)
```

严格标准转移轨道：

```text
n_A_raw = ceil(dva_ref / dv_burn_design)
```

若用户指定总次数：

```text
if maneuver_count_user > 0:
    使用用户指定次数
```

超同步单近地点尾段：

```text
N_total = n_A + 1
```

超同步多近地点尾段：

```text
N_total = n_A + n_P
```

标准转移：

```text
N_total = n_A
```

---

## 12.4 近地点尾段预检查

超同步转移轨道必须估算最后近地点降远地点点火能力。

若假设远地点尾段后：

```text
r_p ≈ r_sync
r_a ≈ 当前超同步远地点
```

最后 P 点火目标：

```text
r_a -> r_sync
```

估算：

```text
dv_P_est = estimate_terminal_perigee_lowering_dv(r_p, r_a, r_sync)
```

若：

```text
tau_P_est <= max_total_burn_time
```

则允许单 P 尾段。

否则：

```text
1. 若 terminal_perigee_mode = multi_P:
       估算 n_P = ceil(dv_P_est / dv_burn_design)
       进入多 P 尾段规划；

2. 若 terminal_perigee_mode = infeasible_if_exceed:
       返回不可行；

3. 若用户允许放宽点火时长:
       输出警告并继续。
```

注意：

```text
增加远地点点火次数 n_A 不一定能降低最后 P 点火 Δv。
```

因为 P 点火主要由最终要降低的远地点高度决定。

# 13. q 序列定义与搜索

## 13.1 q_AA

`q_AA,k` 表示：

```text
第 k 次远地点点火后，飞行到第 q_AA,k 个远地点作为第 k+1 次远地点点火点。
```

取值：

```text
q_AA,k ∈ {1, 2, 3}
```

默认：

```text
q_AA,k = 3
```

---

## 13.2 q_AP

`q_AP` 表示：

```text
最后一次远地点点火后，传播到第 q_AP 个近地点尾段点火机会。
```

定义：

```text
q_AP = 0：二体近似下约半个轨道周期后到达近地点；
q_AP = 1：约 1.5 个周期后到达近地点；
q_AP = 2：约 2.5 个周期后到达近地点。
```

在二体模型中，从远地点到近地点的时间为 `T/2`，与近地点幅角 `ω` 无关。J2 摄动会使真实近地点事件时刻和星下点经度相对二体初值产生偏差，因此最终必须用数值事件搜索确定真实近地点。

---

## 13.3 q 序列搜索策略

q 序列是离散变量，不能放入连续优化器。

V5.0 默认不全枚举所有 q 序列，而采用：

```text
默认 q = 3 + 局部修复 + beam search
```

仅当：

```text
n_A <= 5
```

且 runtime_mode 为 high_quality 时，允许全枚举。

---

## 13.4 q 序列搜索伪代码

```text
function search_q_sequence(n_A, orbit_type, cfg):

    q0 = default_q_sequence(n_A)

    beam = [q0]

    for iter = 1 to max_q_repair_iter:

        candidates = []

        for q_seq in beam:

            result = solve_phase_targets_for_fixed_q(q_seq)

            candidates.append(result)

            if result is feasible:
                continue

            bad_segments = identify_bad_segments(result)

            for seg in bad_segments:

                for q_new in allowed_q_values(seg):

                    q_trial = replace(q_seq, seg, q_new)

                    result_trial = solve_phase_targets_for_fixed_q(q_trial)

                    candidates.append(result_trial)

        beam = select_best_by_score(candidates, beam_width)

        if best candidate is feasible:
            return best candidate

    return best candidate or infeasible
```

默认：

```text
beam_width = 5
max_q_repair_iter = 3
```

# 14. 相位目标变量

## 14.1 连续变量

对固定 q 序列，连续优化变量为：

```text
y = [a_plus,1, a_plus,2, ..., a_plus,n_A-1]
```

或等价：

```text
y = [r_p,plus,1, r_p,plus,2, ..., r_p,plus,n_A-1]
```

二者关系：

```text
a_plus,k = (r_a,k + r_p,plus,k) / 2
```

其中 `r_a,k` 是第 k 次远地点点火位置附近的远地点半径。

推荐内部使用 `a_plus`，输出时同时给出 `r_p,plus`。

---

## 14.2 单调性约束

超同步远地点抬近地点阶段：

```text
r_p,0 <= r_p,plus,1 <= r_p,plus,2 <= ... <= r_sync
```

若采用 `a_plus`：

```text
a_plus,1 <= a_plus,2 <= ... <= a_terminal
```

具体单调方向取决于当前远地点半径。实现中应直接检查 `r_p,plus` 单调增加。

---

## 14.3 解析初值

远地点到远地点相位初值：

```text
Δt_AA,k = q_AA,k * T_k
```

二体周期：

```text
T_k = 2π sqrt(a_plus,k^3 / μ)
```

反解：

```text
a_plus,k ≈ [ μ (Δt_AA,k / (2π q_AA,k))² ]^(1/3)
```

远地点到近地点：

```text
Δt_AP = (q_AP + 0.5) * T
```

反解：

```text
a_plus ≈ [ μ (Δt_AP / (2π (q_AP + 0.5)))² ]^(1/3)
```

以上公式只用于初值。最终必须用 J2 数值传播、拱点事件搜索和经度误差校正。

# 15. 相位评分函数 phase_score

## 15.1 评分原则

V5.0 使用字典序评分元组，而不是单一加权标量，避免不可行方案因为 Δv 小而被误选。

评分元组：

```text
score = (
    invalid,
    window_excess_max,
    terminal_lon_excess,
    terminal_drift_excess,
    terminal_i_excess,
    terminal_a_excess,
    terminal_e_excess,
    duration_excess_max,
    monotonicity_excess,
    dv_spread,
    dv_sum
)
```

含义：

```text
invalid:
    0 表示可传播，1 表示传播失败、求根失败或几何不可行。

window_excess_max:
    点火经度窗口最大超差。

terminal_lon_excess:
    终端平均经度超差。

terminal_drift_excess:
    终端平均经度漂移率超差。

terminal_i_excess:
    终端倾角超差。

terminal_a_excess:
    终端半长轴超差。

terminal_e_excess:
    终端偏心率超差。

duration_excess_max:
    单次点火时长最大超限量。

monotonicity_excess:
    近地点高度单调性违反量。

dv_spread:
    远地点点火 Δv 离散度。

dv_sum:
    总脉冲速度增量。
```

默认容差：

```text
a_tol      = 1.0 km
e_tol      = 1.0e-4
i_tol      = 0.01 deg
lon_tol    = 0.05 deg
drift_tol  = 0.01 deg/day
```

## 15.2 超同步 phase_score 伪代码

```text
function phase_score_supersync(y, q_seq, state_A1, t_A1, cfg):

    invalid = 0
    window_excess_max = 0
    duration_excess_max = 0
    monotonicity_excess = 0
    dv_sum = 0
    dv_list = []

    state = state_A1
    t = t_A1
    mass = m0

    for k = 1 to n_A - 1:

        a_plus_target = y[k]

        # 1. 固定 a_plus，临时使用 alpha 模板或 alpha 初值反求 dv
        dv_k, alpha_k = solve_dv_for_post_a_with_alpha_template(
            state,
            a_plus_target,
            alpha_template
        )

        if solve failed:
            return bad_score

        # 2. 倾角方向规则
        state_trial = apply_impulse(state, dv_k, alpha_k)

        if inclination_error_increases_too_much(state, state_trial):
            invalid = 1

        # 3. 检查近地点高度单调性
        rp_plus = perigee_radius(state_trial)

        monotonicity_excess = max(
            monotonicity_excess,
            monotonicity_violation(rp_plus)
        )

        # 4. 点火时长估算
        tau_k = impulse_dv_to_total_burn_time(dv_k, mass, cfg)

        duration_excess_max = max(
            duration_excess_max,
            tau_k - max_total_burn_time
        )

        # 5. 传播到第 q_AA,k 个远地点
        event_next_A = propagate_to_qth_apsis(
            state_trial,
            t,
            apsis = "A",
            q = q_seq.AA[k]
        )

        if propagation failed:
            return bad_score

        lon_next = sub_satellite_longitude(event_next_A.state, event_next_A.t)

        window_excess_max = max(
            window_excess_max,
            lon_window_excess(lon_next, lon_window_planning)
        )

        dv_sum += dv_k
        dv_list.append(dv_k)

        mass = update_mass_by_impulse(mass, dv_k, cfg)

        state = event_next_A.state
        t = event_next_A.t

    # 6. 最后一次远地点：解析求 rp = r_sync, i = i_target
    dv_n, alpha_n = analytic_solve_apogee_to_rp_i(
        state,
        r_p_target,
        i_target
    )

    if solve failed:
        return bad_score

    state_after_An = apply_impulse(state, dv_n, alpha_n)

    dv_sum += dv_n
    dv_list.append(dv_n)
    mass = update_mass_by_impulse(mass, dv_n, cfg)

    # 7. 传播到近地点尾段
    event_P = propagate_to_qth_apsis(
        state_after_An,
        t,
        apsis = "P",
        q = q_seq.AP
    )

    if propagation failed:
        return bad_score

    lon_P = sub_satellite_longitude(event_P.state, event_P.t)

    window_excess_max = max(
        window_excess_max,
        lon_window_excess(lon_P, lon_window_planning)
    )

    # 8. 近地点降远地点
    if terminal_perigee_mode == single_P:

        dv_P, alpha_P = solve_terminal_perigee_lower_apogee(
            event_P.state,
            r_a_target = r_sync
        )

        state_final = apply_impulse(event_P.state, dv_P, alpha_P)

        dv_sum += dv_P
        dv_list.append(dv_P)

    else if terminal_perigee_mode == multi_P:

        state_final, P_burns = solve_multi_perigee_tail(
            event_P.state,
            event_P.t,
            r_a_target = r_sync
        )

        if failed:
            return bad_score

        append P_burns to dv_list and dv_sum

    # 9. 终端误差
    elem = state_to_orbit_elements(state_final)

    lambda_mean = compute_terminal_mean_longitude(state_final)

    drift_rate = compute_terminal_mean_longitude_drift(state_final)

    terminal_lon_excess = max(
        0,
        abs(wrap180(lambda_mean - lambda_target)) - lon_tol
    )

    terminal_drift_excess = max(
        0,
        abs(drift_rate) - drift_tol
    )

    terminal_i_excess = max(
        0,
        abs(elem.i - i_target) - i_tol
    )

    terminal_a_excess = max(
        0,
        abs(elem.a - a_target) - a_tol
    )

    terminal_e_excess = max(
        0,
        abs(elem.e - e_target) - e_tol
    )

    dv_spread = max(dv_list) - min(dv_list)

    return (
        invalid,
        window_excess_max,
        terminal_lon_excess,
        terminal_drift_excess,
        terminal_i_excess,
        terminal_a_excess,
        terminal_e_excess,
        duration_excess_max,
        monotonicity_excess,
        dv_spread,
        dv_sum
    )
```

# 16. 逐次 alpha 优化

## 16.1 目标

对于第 k 次远地点点火，已知该次点火后的目标半长轴：

```text
a_plus,k,target
```

优化：

```text
alpha_k
```

使：

```text
J_k(alpha) = dv_k(alpha) + dv_remaining_equiv(alpha)
```

最小。

---

## 16.2 当前 dv_k

对给定 alpha：

```text
dv_k(alpha) = solve_dv_for_post_a(
    state_Ak,
    alpha,
    a_plus,k,target
)
```

若无正根，则当前 alpha 不可行。

---

## 16.3 剩余 one-shot dv

执行当前脉冲并传播到下一远地点：

```text
state_next_A = propagate_to_next_selected_apogee(...)
```

在下一远地点计算：

```text
dv_remaining_equiv = analytic_solve_apogee_to_rp_i(
    state_next_A,
    r_p_target = r_sync,
    i_target
)
```

目标：

```text
J_k(alpha) = dv_k + dv_remaining_equiv
```

若点火经度、倾角方向、时长或传播失败，则返回不可行评分。

---

## 16.4 伪代码

```text
function optimize_alpha_for_front_apogee(state_Ak, t_Ak, a_plus_target, q_next, cfg):

    function objective(alpha):

        dv_k = solve_dv_for_post_a(state_Ak, alpha, a_plus_target)

        if failed:
            return bad_score

        state_plus = apply_impulse(state_Ak, dv_k, alpha)

        if inclination_error_increases_too_much(state_Ak, state_plus):
            return bad_score

        event_next_A = propagate_to_qth_apsis(
            state_plus,
            t_Ak,
            apsis = "A",
            q = q_next
        )

        if failed:
            return bad_score

        lon = sub_satellite_longitude(event_next_A.state, event_next_A.t)

        window_excess = lon_window_excess(lon, lon_window_planning)

        dv_rem, alpha_rem = analytic_solve_apogee_to_rp_i(
            event_next_A.state,
            r_p_target,
            i_target
        )

        if failed:
            return bad_score

        return (
            0,
            window_excess,
            dv_k + dv_rem
        )

    alpha_star = scalar_minimize_with_coarse_search_and_refine(objective)

    dv_star = solve_dv_for_post_a(state_Ak, alpha_star, a_plus_target)

    return dv_star, alpha_star
```

# 17. 终端远地点点火

最后一次远地点点火必须完成：

```text
r_p^+ = r_p,target
i^+   = i_target
```

对于超同步转移轨道，这一步之后：

```text
近地点已经抬到同步高度；
倾角已经达到目标；
远地点仍高于同步高度；
后续近地点点火只负责面内降远地点。
```

若此时倾角超差：

```text
|i_plus - i_target| > i_tol
```

则不允许把显著倾角修正留给近地点尾段，应执行回退：

```text
1. 重新求解最后远地点点火，收紧 i_tol 或提高倾角权重；
2. 回退到第 n_A-1 次 alpha 优化，在目标中加入 terminal_i_excess；
3. 重新求解相位目标，给最后远地点保留更多倾角修正能力；
4. 增加远地点点火次数 n_A；
5. 若仍失败，返回不可行。
```

---

# 18. 近地点尾段点火

## 18.1 单 P 点火

在近地点状态 `state_P`，目标：

```text
r_a^+ = r_sync
```

对于近圆同步目标，同时有：

```text
a^+ = r_sync
e^+ = 0
```

点火方向为面内逆速度方向投影：

```text
u_retro = - normalize(v_horizontal)
```

其中：

```text
v_horizontal = v - (v · Rhat) Rhat
```

对应 alpha：

```text
alpha_P = atan2(u_retro · Shat, u_retro · Ehat)
```

然后用固定半长轴二次方程求：

```text
dv_P
```

目标：

```text
a_plus_target = r_sync
```

并检查：

```text
r_a^+ ≈ r_sync
i^+ ≈ i_target
```

---

## 18.2 多 P 点火

若单 P 点火超过能力：

```text
tau_P > max_total_burn_time
```

且允许多 P 模式，则将远地点降低过程分为多段。

假设初始近地点尾段前：

```text
r_p ≈ r_sync
r_a_current > r_sync
```

选择：

```text
n_P = ceil(dv_P_est / dv_burn_design)
```

构造远地点阶段目标：

```text
r_a,0 = 当前远地点半径
r_a,nP = r_sync

r_a,j = r_sync + (r_a,0 - r_sync) * (1 - j/n_P)
```

第 j 次近地点点火后目标半长轴：

```text
a_plus,j = (r_sync + r_a,j) / 2
```

每次 P 点火后传播到下一次满足经度窗口的近地点，继续降远地点。

结构：

```text
A_1, A_2, ..., A_n, P_1, P_2, ..., P_m
```

其中：

```text
m = n_P
```

多 P 模式也需要满足：

```text
1. 每次 P 点火经度在窗口内；
2. 相邻 P 点火飞行圈次 <= max_q_between_burns；
3. 每次 P 点火时长不超限；
4. 最后终端平均经度和漂移率满足要求。
```

# 22. 脉冲到有限推力初值转换

## 22.1 主发动机时长

给定脉冲 Δv：

```text
m_s = m_b - mdot_set * tau_set
dv_set = Isp_set * g0 * ln(m_b / m_s)
dv_main = max(0, dv_impulse - dv_set)
```

主发动机点火时长：

```text
tau_main = m_s / mdot_main_eff * [1 - exp(-dv_main / c_main_eff)]
```

---

## 22.2 初始对齐

默认将脉冲时刻作为主发动机弧段中心：

```text
ts = T_impulse - tau_main / 2
te = T_impulse + tau_main / 2
tb = ts - tau_set
```

其中：

```text
tb = 沉底开始
ts = 主机开始
te = 主机结束
```

---

## 22.3 弧段中心迭代修正

对于 60~90 min 的长点火，脉冲中心对齐可能产生明显误差。因此建议增加弧段中心迭代修正。

简化迭代：

```text
function recenter_finite_burn(burn, X_pre, cfg):

    T_center = burn.T_impulse

    for iter = 1 to max_recenter_iter:

        ts = T_center - tau_main / 2
        te = T_center + tau_main / 2
        tb = ts - tau_set

        X_finite = propagate_settling_and_main_burn(
            X_pre,
            tb,
            ts,
            te,
            alpha
        )

        error = compare_to_impulse_reference(X_finite)

        if error < tolerance:
            break

        T_center = T_center + center_time_correction(error)

    return tb, ts, te
```

也可使用经度误差修正：

```text
T_center_new = T_center - k_lon * wrap180(lambda_finite - lambda_impulse)
```

该步骤用于生成更好的有限推力初值，不替代后续有限推力优化。

# 23. 约束检查

## 23.1 经度窗口

每次脉冲点：

```text
lambda_ss(T_k) ∈ lon_window_planning
```

转有限推力后，建议检查：

```text
lambda_ss(tb,k)
lambda_ss(ts,k)
lambda_ss(te,k)
```

若要求整个弧段都在测控窗口内，应每隔 60~120 s 采样检查。

---

## 23.2 点火时长

```text
tau_total,k = tau_set + tau_main,k
```

要求：

```text
tau_total,k <= max_total_burn_time
```

若任务只限制主发动机，则使用：

```text
tau_main,k <= max_main_burn_time
```

---

## 23.3 终端轨道

终端误差：

```text
da = a_f - a_target
de = e_f - e_target
di = i_f - i_target
dlon = wrap180(lambda_mean_f - lambda_target)
ddrift = dot_lambda_mean_f
```

默认约束：

```text
|da| <= 1 km
|de| <= 1e-4
|di| <= 0.01 deg
|dlon| <= 0.05 deg
|ddrift| <= 0.01 deg/day
```

---

## 23.4 一致性检查

远地点总 Δv 一致性：

```text
rho_A = sum(dv_A_1 ... dv_A_n) / dva_ref
```

默认工程经验阈值：

```text
0.95 <= rho_A <= 1.10       正常
1.10 < rho_A <= 1.15        警告
rho_A > 1.15                异常，需要检查 q_seq 或相位目标
```

该阈值是工程经验检查值，不是理论硬约束。不同初始轨道、经度窗口、q 序列和摄动模型下可调整。

|---|---|
| 前段 alpha 一维优化 | coarse grid + Brent |
| 固定 a_plus 反解 dv | 二次方程 |
| 目标 rp 求 dv | Brent / bracket root |
| 相位目标 y 优化 | coordinate search / SLSQP |
| q 序列 | beam search / local repair |
| 整体小范围精修 | SLSQP / coordinate search |

---

## 24.3 缓存

建议启用缓存：

```text
1. 拱点传播缓存：
   key = state_hash + apsis_type + q

2. 固定半长轴反解缓存：
   key = state_hash + a_plus_target + alpha

3. one-shot 剩余 dv 缓存：
   key = state_hash + r_p_target + i_target

4. q_seq 评分缓存：
   key = q_seq + rounded_a_plus_sequence
```

```text

2. q_seq beam 中不同候选；
3. phase_score 中多个候选 y；

```

若用于 UI 软件，应放入 worker 线程，主界面只显示进度和当前最佳解。

## 27.2 派生量

采用：

```text
Re = 6378.137 km
r_sync = 42164.2 km
```

近地点半径：

```text
r_p0 = a(1-e)
     = 29478.137 * (1 - 0.77684692)
     ≈ 6578.137 km
```

近地点高度：

```text
h_p0 = r_p0 - Re
     ≈ 200.000 km
```

远地点半径：

```text
r_a0 = a(1+e)
     = 29478.137 * (1 + 0.77684692)
     ≈ 52378.137 km
```

远地点高度：

```text
h_a0 = r_a0 - Re
     ≈ 46000.000 km
```

同步高度：

```text
h_sync = r_sync - Re
       = 42164.2 - 6378.137
       ≈ 35786.063 km
```

远地点高度超同步量：

```text
h_a0 - h_sync
≈ 46000.000 - 35786.063
≈ 10213.937 km
```

因此该算例判定为：

```text
supersynchronous_transfer
```

推荐基础结构为：

```text
A, A, ..., A, P
```

若最后 P 点火时长超限，则结构扩展为：

```text
A, A, ..., A, P, P, ...
```

# 29. 风险与处理

| 风险 | 表现 | 处理 |
|---|---|---|
| q 序列不可行 | 找不到经度窗口内下一拱点 | q 局部修复、beam search、增加次数 |
| P 点火超限 | 最后近地点点火过长 | 多 P 尾段或判不可行 |
| 标准转移几何不可达 | 末次 A 后不能 e=0 | strict/near standard 区分 |
| alpha 贪心局部最优 | 每步好但总解差 | 整体小范围精修 |
| 终端平均经度不稳 | e 接近 0 时 ω/M 奇异 | 非奇异平均经度 |
| 有限推力弧段越界 | 脉冲点在窗口内但弧段出界 | 收缩窗口 + 弧段中心修正 |
| 计算量过大 | UI 卡顿 | worker 线程、缓存、time budget |

# 30. 结论

V5.0 是一套相位约束驱动的序贯脉冲规划算法。它以远地点/近地点拱点结构为基础，通过 q 序列控制点火间隔，通过每次点火后的半长轴或近地点高度控制相位，通过逐次 alpha 优化降低局部求解维度，并通过整体小范围精修消除贪心误差。

超同步转移轨道的主结构为：

```text
n 次远地点点火 + 1 次或多次近地点点火
```

标准转移轨道的主结构为：

```text
n 次远地点点火
```

但标准转移必须区分严格标准与近标准，否则终端圆化存在几何不可达风险。

该算法适合作为有限推力序贯优化的脉冲初始规划器。实现时应重点保证：

```text
1. q_seq 离散搜索与 a_plus 连续优化分离；
2. phase_score 完整可执行；
3. 终端经度使用平均经度；
4. P 点火超限有多 P 或不可行处理；
5. alpha 优化后必须整体精修；
6. 有限推力转换时进行弧段中心修正。
```
