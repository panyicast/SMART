# 五次脉冲点火任务规划算法文档

```text
远地点回归约束 · 三圈内可再次点火 · J2 摄动 · 经度窗口 · 沉底推力器 · 姿控效率修正 · 推进剂最小化
```

版本：V1.0  
适用任务：从初始高椭圆轨道出发，经五次脉冲初始规划生成有限推力序贯优化的点火初值，最终转入标称经度 120°E、倾角 6° 的同步轨道。

---

# 1. 文档目的

本文档给出一套**完整的五次脉冲点火任务规划算法**，用于为后续有限推力序贯优化提供可靠初值。

本算法的核心思想不是在全时间轴上任意搜索五个点火时刻，而是采用**远地点回归约束驱动的事件搜索方法**：

1. 从初始轨道自然传播，寻找最近一个星下点经度满足点火窗口的远地点，作为第一次变轨的大致时间点；
2. 每次点火后，控制量必须保证卫星在最多飞行 3 圈以内，其远地点重新进入可点火经度窗口；
3. 下一次点火时间由上一次点火后的远地点回归事件确定，而不是作为独立自由变量随意搜索；
4. 在满足远地点回归、经度窗口、单次点火时间、终端轨道等约束的基础上，再最小化等效推进剂消耗。

本算法只使用以下先验信息：

- 初始轨道；
- 初始质量；
- 目标轨道；
- 发动机参数；
- 沉底推力器参数；
- 点火经度窗口；
- J2 摄动模型；
- 地球自转模型；
- 最大三圈内远地点回归要求。

算法**不得使用最终有限推力优化结果**作为参考数据，例如不得使用最终优化后的点火时间、速度增量、推进剂消耗、中间轨道状态或点火经度作为先验初值。

## 2.3 主发动机与沉底推力器参数

| 参数 | 符号 | 数值 |
|---|---:|---:|
| 主发动机推力 | `F_main` | 490 N |
| 主发动机名义比冲 | `Isp_main` | 314.1 s |
| 标准重力加速度 | `g0` | 9.80665 m/s² |
| 姿控效率修正 | `eta_ac` | 1.73% = 0.0173 |
| 沉底推力器推力 | `F_set` | 20 N |
| 沉底推力器比冲 | `Isp_set` | 290 s |
| 每次沉底时长 | `tau_set` | 240 s |
| 单次主机最大点火时长 | `tau_main_max` | 5400 s = 90 min |

主发动机等效比冲：

```text
Isp_main_eff = Isp_main / (1 + eta_ac)
```

代入数值：

```text
Isp_main_eff = 314.1 / 1.0173 ≈ 308.758 s
```

主发动机等效喷速：

```text
c_main_eff = Isp_main_eff * g0
```

主发动机等效质量流率：

```text
mdot_main_eff = F_main / c_main_eff
```

沉底推力器喷速：

```text
c_set = Isp_set * g0
```

沉底推力器质量流率：

```text
mdot_set = F_set / c_set
```

每次沉底推进剂消耗：

```text
m_set = mdot_set * tau_set
```

# 3. 约束条件总览

## 3.1 点火经度窗口

工程要求：

```text
40°E <= lambda <= 180°E
```

为了给后续有限推力弧段展开留裕度，脉冲初始规划阶段建议使用收缩窗口：

```text
50°E <= lambda <= 170°E
```

其中：

- `[40°E, 180°E]` 为最终硬约束检查窗口；
- `[50°E, 170°E]` 为脉冲初始规划推荐窗口；
- 若收缩窗口下难以找到可行解，可先放宽到 `[45°E, 175°E]`，但最终必须回到 `[40°E, 180°E]` 检查。

---

## 3.2 远地点回归约束

这是本算法的核心约束。

第 `k` 次点火后，令点火后状态为：

```text
X_k^+ = [r_k^+, v_k^+]
```

从 `X_k^+` 开始无推力传播，检测未来 3 个远地点：

```text
A_{k,1}, A_{k,2}, A_{k,3}
```

其中：

- `A_{k,1}`：点火后第 1 个远地点；
- `A_{k,2}`：点火后第 2 个远地点；
- `A_{k,3}`：点火后第 3 个远地点。

要求至少存在一个远地点满足点火窗口：

```text
exists q ∈ {1,2,3}, such that lambda(A_{k,q}) ∈ [50°E, 170°E]
```

或者在最终硬约束版本中：

```text
exists q ∈ {1,2,3}, such that lambda(A_{k,q}) ∈ [40°E, 180°E]
```

若第 `k` 次点火后 3 圈内没有远地点回到点火窗口，则该控制量不可行。

该约束适用于：

```text
k = 1, 2, 3, 4
```

第 5 次点火后不再要求回归，因为第 5 次直接满足终端轨道约束。

---

## 3.3 点火位置约束

本任务的脉冲中心时刻默认位于**远地点附近**。

第 `k` 次点火中心时刻写成：

```text
T_k = t_A,k + delta_t_k
```

其中：

- `t_A,k` 为第 `k` 次选定远地点事件时刻；
- `delta_t_k` 为远地点附近的小时间偏移。

约束：

```text
|delta_t_k| <= delta_t_A,max
```

建议：

```text
delta_t_A,max = 10 min ~ 30 min
```

也可用真近点角约束：

```text
|wrap180(nu_k - 180°)| <= Delta_nu_A
```

建议：

```text
Delta_nu_A = 5° ~ 20°
```

---

## 3.4 单次点火时间约束

脉冲规划本身是瞬时速度增量模型，但每个脉冲必须能够展开为有限推力弧段。

第 `k` 次脉冲的等效主发动机工作时间估算为：

```text
tau_main,k = m_s,k / mdot_main_eff * [1 - exp(-DeltaV_main,k / c_main_eff)]
```

其中：

```text
m_s,k = m_k - m_set
DeltaV_set,k = c_set * ln(m_k / m_s,k)
DeltaV_main,k = max(0, DeltaV_k - DeltaV_set,k)
```

要求：

```text
tau_main,k <= 5400 s
```

如果任务要求沉底段和主发动机段合计不超过 90 min，则改为：

```text
tau_set + tau_main,k <= 5400 s
```

本算法默认采用：

```text
主发动机工作时间 <= 90 min
```

# 4. 坐标系与点火方向

## 4.1 状态变量

采用地心赤道惯性系 ECI：

```text
X = [x, y, z, vx, vy, vz]^T
```

位置、速度单位建议采用：

```text
r: km
v: km/s
```

质量单独记为：

```text
m: kg
```

---

## 4.2 当地水平坐标系

令：

```text
R_hat = r / |r|
K_hat = [0, 0, 1]^T
```

当地正东方向：

```text
E_hat = K_hat × R_hat / |K_hat × R_hat|
```

当地正北方向：

```text
N_hat = [K_hat - (K_hat · R_hat) R_hat] / |K_hat - (K_hat · R_hat) R_hat|
```

当地正南方向：

```text
S_hat = -N_hat
```

---

## 4.3 点火方向角 alpha

第 `k` 次脉冲方向角 `alpha_k` 定义为：

```text
从当地正东方向起算，向当地正南方向为正。
```

| `alpha_k` | 含义 |
|---:|---|
| 0° | 当地正东方向 |
| 90° | 当地正南方向 |
| -90° | 当地正北方向 |
| 180° 或 -180° | 当地正西方向 |

脉冲方向单位向量：

```text
u_k = cos(alpha_k) E_hat + sin(alpha_k) S_hat
```

该定义天然满足无径向分量：

```text
r · u_k = 0
```

脉冲速度更新：

```text
v_k^+ = v_k^- + DeltaV_k * u_k
```

位置不变：

```text
r_k^+ = r_k^-
```

---

# 5. 动力学模型

## 5.1 脉冲间 J2 摄动传播

脉冲之间采用无推力动力学：

```text
r_ddot = -mu * r / |r|^3 + a_J2
```

J2 摄动加速度：

```text
a_J2 = (3 J2 mu Re^2 / 2 r^5) *
       [ x(5z^2/r^2 - 1),
         y(5z^2/r^2 - 1),
         z(5z^2/r^2 - 3) ]^T
```

建议常数：

| 常数 | 符号 | 数值 |
|---|---:|---:|
| 地球引力常数 | `mu` | 398600.4418 km³/s² |
| 地球赤道半径 | `Re` | 6378.137 km |
| J2 项 | `J2` | 1.08262668e-3 |
| 地球自转角速度 | `omega_e` | 7.2921158553e-5 rad/s |

---

## 5.2 粗搜索与精修模型分级

为了兼顾速度和精度，建议分两级实现：

| 阶段 | 模型 | 用途 |
|---|---|---|
| 粗搜索 | J2 平均根数传播或大步长 Cowell 积分 | 快速筛选候选解 |
| 局部精修 | Cowell + J2 高精度数值积分 | 精确闭合终端轨道 |

推荐高精度积分器：

```text
DOP853 / RK78 / Dormand-Prince 8(7) / Radau
```

点火事件搜索时必须强制积分到远地点事件。

---

# 6. 星下点经度与终端经度

## 6.1 星下点经度

ECI 位置转换到 ECEF：

```text
r_ECEF = R3[-theta_G(t)] * r_ECI
```

星下点经度：

```text
lambda = atan2(y_ECEF, x_ECEF)
```

统一转换为：

```text
0° <= lambda < 360°
```

东经为正。

---

## 6.2 标称经度

终端倾斜同步轨道的标称经度可采用：

```text
lambda_nom = Omega + u - theta_G(t)
```

其中：

- `Omega` 为升交点赤经；
- `u` 为纬度幅角；
- `theta_G(t)` 为格林尼治恒星时角。

终端经度约束：

```text
wrap180(lambda_nom - 120°) = 0
```

---

# 7. 远地点事件检测

## 7.1 径向速度

定义径向速度：

```text
v_r = r · v / |r|
```

远地点事件满足：

```text
v_r = 0
```

且径向速度符号变化为：

```text
v_r: positive -> negative
```

即卫星由远离地球转为接近地球。

---

## 7.2 远地点事件数据结构

每个远地点事件保存为：

```text
A_j = {
    event_id,
    orbit_count,
    t_A,
    X_A,
    lambda_A,
    a_A,
    e_A,
    i_A,
    omega_A,
    Omega_A,
    nu_A
}
```

其中：

```text
nu_A ≈ 180°
```

---

## 7.3 远地点附近窗口

对每个远地点事件，建立局部点火窗口：

```text
W_A,j = [t_A,j - delta_t_A,max, t_A,j + delta_t_A,max]
```

实际点火中心：

```text
T_k = t_A,j + delta_t_k
```

其中：

```text
|delta_t_k| <= delta_t_A,max
```

---

# 8. 第一次点火时间搜索

## 8.1 自然传播寻找远地点

从初始状态 `X0` 出发，不施加控制，进行 J2 无推力传播，检测远地点事件序列：

```text
A_1, A_2, A_3, ...
```

每个远地点计算星下点经度：

```text
lambda_A,j
```

---

## 8.2 最近可点火远地点

第一次点火远地点圈号定义为：

```text
j_1 = min j, such that lambda_A,j ∈ [lambda_min_plan, lambda_max_plan]
```

其中脉冲规划推荐窗口为：

```text
lambda_min_plan = 50°E
lambda_max_plan = 170°E
```

若在收缩窗口中找不到远地点，可按以下顺序处理：

1. 放宽到 `[45°E, 175°E]`；
2. 再放宽到工程硬窗口 `[40°E, 180°E]`；
3. 若仍找不到，则延长自然传播搜索时长。

第一次点火初始时刻：

```text
T_1 = t_A,j1 + delta_t_1
```

其中：

```text
|delta_t_1| <= delta_t_A,max
```

---

# 9. 后续点火时间递推逻辑

## 9.1 点火后搜索未来 3 个远地点

第 `k` 次点火后，得到状态：

```text
X_k^+
```

从 `X_k^+` 出发无推力传播，依次检测未来 3 个远地点：

```text
A_{k,1}, A_{k,2}, A_{k,3}
```

计算它们的经度：

```text
lambda_{k,1}, lambda_{k,2}, lambda_{k,3}
```

定义可回归集合：

```text
Q_k = { q | q ∈ {1,2,3}, lambda_{k,q} ∈ [lambda_min_plan, lambda_max_plan] }
```

若：

```text
Q_k = empty
```

则第 `k` 次点火控制量不可行。

---

## 9.2 下一次点火由回归远地点确定

若：

```text
Q_k ≠ empty
```

则第 `k+1` 次点火远地点可从 `Q_k` 中选择：

```text
q_k ∈ Q_k
```

下一次点火事件：

```text
t_A,k+1 = t(A_{k,q_k})
X_A,k+1 = X(A_{k,q_k})
```

下一次脉冲中心时刻：

```text
T_{k+1} = t_A,k+1 + delta_t_{k+1}
```

因此，第 `k+1` 次点火时间不是独立自由变量，而是由第 `k` 次点火后的轨道回归决定。

# 10. 优化变量

## 10.1 不推荐的变量形式

不建议直接优化五个绝对时间：

```text
Z = [T1, DeltaV1, alpha1, ..., T5, DeltaV5, alpha5]
```

原因是 `T2 ~ T5` 与前一次点火后的轨道周期、漂移率和远地点经度直接相关，不应独立自由搜索。

---

## 10.2 推荐变量形式

外层离散变量：

```text
q = [q1, q2, q3, q4]
```

其中：

```text
q_k ∈ {1, 2, 3}
```

表示第 `k` 次点火后，选择第 `q_k` 个远地点作为下一次点火远地点。

内层连续变量：

```text
y = [
    delta_t1, DeltaV1, alpha1,
    delta_t2, DeltaV2, alpha2,
    delta_t3, DeltaV3, alpha3,
    delta_t4, DeltaV4, alpha4,
    delta_t5, DeltaV5, alpha5
]
```

其中：

| 变量 | 含义 |
|---|---|
| `delta_t_k` | 相对选定远地点的小时间偏移 |
| `DeltaV_k` | 第 k 次脉冲速度增量 |
| `alpha_k` | 当地水平面内脉冲方向角 |

外层最多枚举：

```text
3^4 = 81
```

组回归圈数序列。

# 11. 推进剂等效模型

## 11.1 脉冲速度增量定义

第 `k` 次脉冲变量 `DeltaV_k` 表示本次机动的等效总速度增量，包括：

```text
沉底推力器贡献 + 主发动机贡献
```

脉冲动力学更新使用总速度增量：

```text
v_k^+ = v_k^- + DeltaV_k * u_k
```

---

## 11.2 本次推进剂计算

第 `k` 次点火前质量为 `m_k`。

沉底推进剂：

```text
m_set = mdot_set * tau_set
```

沉底后质量：

```text
m_s,k = m_k - m_set
```

沉底等效速度增量：

```text
DeltaV_set,k = c_set * ln(m_k / m_s,k)
```

主发动机需要补足的速度增量：

```text
DeltaV_main,k = max(0, DeltaV_k - DeltaV_set,k)
```

主发动机工作后质量：

```text
m_{k+1} = m_s,k * exp(-DeltaV_main,k / c_main_eff)
```

本次推进剂消耗：

```text
m_prop,k = m_k - m_{k+1}
```

总推进剂消耗：

```text
J_prop = Σ_{k=1..5} m_prop,k = m0 - m6
```

---

## 11.3 主发动机点火时长估算

```text
tau_main,k = m_s,k / mdot_main_eff * [1 - exp(-DeltaV_main,k / c_main_eff)]
```

约束：

```text
tau_main,k <= tau_main_max
```

若违反，则加罚或判为不可行。

---

# 12. 目标函数

总目标函数：

```text
J = J_prop
  + W_terminal * P_terminal
  + W_return   * P_return
  + W_window   * P_window
  + W_apsis    * P_apsis
  + W_tau      * P_tau
```

其中：

- `J_prop`：等效推进剂消耗；
- `P_terminal`：终端轨道误差罚函数；
- `P_return`：三圈内远地点回归罚函数；
- `P_window`：点火经度窗口罚函数；
- `P_apsis`：远地点附近点火罚函数；
- `P_tau`：主发动机点火时长罚函数。

---

## 12.1 终端轨道罚函数

终端轨道目标：

```text
a_target = 42164.2 km
e_target = 0
i_target = 6°
lambda_target = 120°E
lambda_dot_target = 0
```

罚函数：

```text
P_terminal =
    [(a_f - a_target) / sigma_a]^2
  + [e_f / sigma_e]^2
  + [(i_f - i_target) / sigma_i]^2
  + [wrap180(lambda_f - lambda_target) / sigma_lambda]^2
  + [lambda_dot_f / sigma_drift]^2
```

建议归一化尺度：

| 项 | 粗搜索 | 精修 |
|---|---:|---:|
| `sigma_a` | 100 km | 1 km |
| `sigma_e` | 1e-2 | 5e-4 |
| `sigma_i` | 0.5° | 0.01° |
| `sigma_lambda` | 10° | 0.2° |
| `sigma_drift` | 按任务设定 | 按任务设定 |

---

## 12.2 三圈回归罚函数

对 `k = 1,2,3,4`，第 `k` 次点火后未来 3 个远地点的经度为：

```text
lambda_{k,1}, lambda_{k,2}, lambda_{k,3}
```

定义到窗口的距离：

```text
d(lambda, [lambda_min, lambda_max]) =
    0,                         if lambda_min <= lambda <= lambda_max
    lambda_min - lambda,       if lambda < lambda_min
    lambda - lambda_max,       if lambda > lambda_max
```

连续罚函数：

```text
P_return,k = min_{q=1,2,3} d(lambda_{k,q}, [lambda_min_plan, lambda_max_plan])^2
```

硬约束版本：

```text
if P_return,k > 0:
    infeasible
```

推荐策略：

- 粗搜索阶段使用连续罚函数；
- 精修阶段使用硬约束。

---

## 12.3 点火经度窗口罚函数

第 `k` 次点火中心时刻经度：

```text
lambda_k = lambda(T_k)
```

罚函数：

```text
P_window,k = d(lambda_k, [lambda_min_plan, lambda_max_plan])^2
```

有限推力展开检查时，还应估算：

```text
tb,k = T_k - tau_main,k/2 - tau_set
nts,k = T_k - tau_main,k/2
te,k = T_k + tau_main,k/2
```

并检查：

```text
lambda(tb,k), lambda(ts,k), lambda(te,k) ∈ [40°E, 180°E]
```

> 注：上式中的 `nts,k` 表示主发动机开始时刻，即通常记为 `ts,k`。若代码实现中使用 `ts` 作为变量名，文档中的 `nts,k` 可直接改为 `ts,k`。

---

## 12.4 远地点偏离罚函数

使用真近点角：

```text
P_apsis,k = [wrap180(nu_k - 180°) / sigma_nu]^2
```

或者使用径向速度：

```text
P_apsis,k = (v_r,k / sigma_vr)^2
```

推荐组合：

```text
P_apsis,k = W_nu [wrap180(nu_k - 180°) / sigma_nu]^2
            + W_vr (v_r,k / sigma_vr)^2
```

建议：

```text
sigma_nu = 5° ~ 10°
sigma_vr = 0.01 km/s
```

---

## 12.5 点火时长罚函数

```text
P_tau,k = max(0, tau_main,k - tau_main_max)^2
```

精修阶段建议使用硬约束：

```text
tau_main,k <= 5400 s
```

# 13. 搜索与优化流程

## 13.1 总体流程

```text
输入任务参数
  ↓
初始轨道自然传播
  ↓
寻找最近满足经度窗口的远地点，确定第 1 次点火事件
  ↓
枚举第 1~4 次点火后的回归圈数 q_k ∈ {1,2,3}
  ↓
对每组 q 序列，连续优化 delta_t_k, DeltaV_k, alpha_k
  ↓
每次点评估：点火 → 未来 3 个远地点回归检查 → 递推下一次点火事件
  ↓
第 5 次点火后计算终端轨道误差
  ↓
选择满足约束且推进剂最小的五次脉冲规划结果
  ↓
转换为有限推力初值
```

---

## 13.2 第一步：寻找第一次远地点

```text
function find_first_feasible_apogee(X0, t0):
    X = X0
    t = t0

    while t < t0 + T_search_max:
        propagate with J2
        detect apogee event A_j
        compute lambda_A,j

        if lambda_A,j in [lambda_min_plan, lambda_max_plan]:
            return A_j

    return failure
```

输出：

```text
A_first = {
    t_A,1,
    X_A,1,
    lambda_A,1,
    orbit_count_1
}
```

第一次点火初值：

```text
T1 = t_A,1
```

连续优化时使用：

```text
T1 = t_A,1 + delta_t1
```

---

## 13.3 第二步：枚举回归圈数序列

```text
for q1 in {1,2,3}:
  for q2 in {1,2,3}:
    for q3 in {1,2,3}:
      for q4 in {1,2,3}:
          q_seq = [q1, q2, q3, q4]
          solve_continuous_optimization(q_seq)
```

每组 `q_seq` 表示：

```text
第 1 次点火后，选择第 q1 个远地点作为第 2 次点火事件；
第 2 次点火后，选择第 q2 个远地点作为第 3 次点火事件；
第 3 次点火后，选择第 q3 个远地点作为第 4 次点火事件；
第 4 次点火后，选择第 q4 个远地点作为第 5 次点火事件。
```

如果某一次选择的第 `q_k` 个远地点不满足经度窗口，则该 `q_seq` 在当前连续变量下不可行。

---

## 13.4 第三步：连续变量优化

对于固定的 `q_seq`，优化变量为：

```text
y = [delta_t1, DeltaV1, alpha1,
     delta_t2, DeltaV2, alpha2,
     delta_t3, DeltaV3, alpha3,
     delta_t4, DeltaV4, alpha4,
     delta_t5, DeltaV5, alpha5]
```

建议变量边界：

| 变量 | 推荐边界 |
|---|---:|
| `delta_t_k` | `[-30 min, +30 min]` |
| `DeltaV_k` | `[0, DeltaV_max,k]` |
| `alpha_k` | `[-180°, 180°]` |

其中 `DeltaV_max,k` 由当前质量和 90 min 点火时长估计：

```text
DeltaV_main,max,k = c_main_eff * ln[m_s,k / (m_s,k - mdot_main_eff * tau_main_max)]
DeltaV_max,k = DeltaV_set,k + DeltaV_main,max,k
```

优化器建议：

| 阶段 | 推荐方法 |
|---|---|
| 粗搜索 | Differential Evolution、CMA-ES、Particle Swarm、Pattern Search |
| 精修 | SLSQP、SNOPT、IPOPT、trust-constr |
| 最终校验 | 高精度 Cowell + J2 积分 |

# 14. 单组 q 序列的评价函数

## 14.1 评价函数输入

```text
q_seq = [q1, q2, q3, q4]
y = [delta_t1, DeltaV1, alpha1, ..., delta_t5, DeltaV5, alpha5]
```

---

## 14.2 评价函数流程

```text
function evaluate(q_seq, y):

    X_event = A_first.X
    t_event = A_first.t
    m = m0
    J_prop = 0
    history = []

    for k = 1 to 5:

        delta_t_k = y[3k-2]
        DeltaV_k  = y[3k-1]
        alpha_k   = y[3k]

        T_k = t_event + delta_t_k

        propagate X_event from t_event to T_k
        obtain X_k_minus

        compute lambda_k, nu_k, v_r,k
        add apogee-deviation penalty
        add ignition-longitude penalty

        compute local horizontal frame E_hat, S_hat
        u_k = cos(alpha_k) E_hat + sin(alpha_k) S_hat

        apply impulse:
            v_k_plus = v_k_minus + DeltaV_k * u_k
            X_k_plus = [r_k_minus, v_k_plus]

        compute equivalent propellant:
            m_s = m - m_set
            DeltaV_set = c_set * ln(m / m_s)
            DeltaV_main = max(0, DeltaV_k - DeltaV_set)
            m_next = m_s * exp(-DeltaV_main / c_main_eff)
            m_prop = m - m_next
            tau_main = m_s / mdot_main_eff * [1 - exp(-DeltaV_main / c_main_eff)]

        add tau_main penalty if tau_main > 5400 s

        J_prop += m_prop
        m = m_next

        if k < 5:
            propagate X_k_plus and detect next 3 apogees:
                A_1, A_2, A_3

            qk = q_seq[k]
            A_selected = A_qk

            if lambda(A_selected) not in planning window:
                return large_penalty

            t_event = A_selected.t
            X_event = A_selected.X

        else:
            compute terminal orbital elements:
                a_f, e_f, i_f, lambda_f, lambda_dot_f
            compute terminal penalty

    return J_prop + penalties
```

# 16. 有限推力初值转换

脉冲规划完成后，需要转换为有限推力序贯优化初值。

第 `k` 次脉冲中心时刻：

```text
T_k
```

对应主发动机估算工作时间：

```text
tau_main,k
```

主发动机开始、结束时刻：

# 19. 推荐实现伪代码

```text
输入：
    X0, m0, t0
    目标轨道 a=42164.2 km, e≈0, i=6°, lambda=120°E
    经度窗口 [40°E, 180°E]
    规划收缩窗口 [50°E, 170°E]
    N_return = 3
    发动机参数 F_main, Isp_main, eta_ac=0.0173
    沉底参数 F_set, Isp_set, tau_set=240 s

预处理：
    Isp_main_eff = Isp_main / (1 + eta_ac)
    c_main_eff = Isp_main_eff * g0
    mdot_main_eff = F_main / c_main_eff

    c_set = Isp_set * g0
    mdot_set = F_set / c_set
    m_set = mdot_set * tau_set

步骤 1：寻找第一次可点火远地点
    A_first = find_first_feasible_apogee(X0, t0)
    if A_first not found:
        调整搜索时长或放宽规划窗口

步骤 2：枚举回归圈数序列
    best_solution = None

    for q_seq in all combinations of [1,2,3]^4:

        生成连续变量初值 y0

        result = optimize evaluate(q_seq, y)

        if result feasible and result.J < best_solution.J:
            best_solution = result

步骤 3：评价函数 evaluate(q_seq, y)

    X_event = A_first.X
    t_event = A_first.t
    m = m0
    J_prop = 0

    for k = 1 to 5:

        读取 delta_t_k, DeltaV_k, alpha_k
        T_k = t_event + delta_t_k

        从 X_event, t_event 传播到 T_k，得到 X_k_minus

        计算：
            lambda_k
            nu_k
            v_r,k

        检查：
            远地点附近性
            点火中心经度

        计算当地水平面方向 u_k

        施加脉冲：
            X_k_plus.v = X_k_minus.v + DeltaV_k * u_k
            X_k_plus.r = X_k_minus.r

        计算推进剂消耗与主机时长：
            m_s = m - m_set
            DeltaV_set = c_set * ln(m / m_s)
            DeltaV_main = max(0, DeltaV_k - DeltaV_set)
            m_next = m_s * exp(-DeltaV_main / c_main_eff)
            tau_main = m_s / mdot_main_eff * [1 - exp(-DeltaV_main/c_main_eff)]

        若 tau_main > 5400 s：
            加罚或不可行

        J_prop += m - m_next
        m = m_next

        if k < 5:
            从 X_k_plus 开始传播，检测未来 3 个远地点
            apogees = detect_next_apogees(X_k_plus, T_k, N=3)

            qk = q_seq[k]
            A_selected = apogees[qk]

            if lambda(A_selected) 不在规划窗口：
                返回不可行或加大罚值

            X_event = A_selected.X
            t_event = A_selected.t

        else:
            计算终端轨道参数
            计算 P_terminal

    返回：
        J = J_prop + P_terminal + 其他罚函数

步骤 4：输出
    五次脉冲时刻 T_k
    DeltaV_k
    alpha_k
    每次推进剂估算
    每次点火后 3 圈回归检查
    有限推力初值 tb, ts, te
    终端轨道误差
```

# 20. 工程实现建议

## 20.1 先硬后软的约束处理顺序

建议按以下顺序筛选：

1. 第一次远地点是否满足经度窗口；
2. 每次 `DeltaV_k` 是否可展开为 90 min 内有限推力；
3. 每次点火后 3 圈内是否存在可点火远地点；
4. 第五次点火后终端轨道是否接近目标；
5. 在可行解中比较等效推进剂。

不要过早只优化推进剂，否则容易得到后续无法回归窗口的局部解。

---

## 20.2 采用 Beam Search 提高效率

如果直接优化 81 个 `q_seq` 较慢，可以使用 Beam Search：

```text
每一层只保留推进剂较低、终端趋势较好、回归窗口裕度较大的前 B 个节点。
```

推荐：

```text
B = 10 ~ 30
```

节点评分：

```text
score = 当前推进剂消耗
      + 当前轨道到目标轨道的粗略误差
      + 远地点回归窗口裕度罚函数
```

---

## 20.3 多初值策略

对每个固定 `q_seq`，应使用多组连续初值：

```text
DeltaV_k: 由轨道能量差估算并随机扰动
alpha_k: 从离散方向集合采样
    {-90°, -60°, -30°, 0°, 30°, 60°, 90°}
delta_t_k:
    {-20 min, -10 min, 0, 10 min, 20 min}
```

注意：上述方向和时间偏移只是搜索初值，不来自最终优化结果。

---

## 20.4 粗搜索到精修的权重调整

粗搜索阶段：

```text
经度回归罚函数、点火时长罚函数、远地点偏离罚函数权重大；
终端轨道误差允许较大；
```

精修阶段：

```text
逐渐增大终端轨道罚函数权重；
将三圈回归和点火时长改为硬约束；
将积分器换成高精度 Cowell + J2。
```

---

# 21. 可选扩展：近地点点火模式

本版算法默认每次脉冲中心位于远地点附近，因为任务逻辑要求每次点火后最多 3 圈内远地点回归可点火区域。

若后续任务允许部分近地点点火，可扩展事件类型：

```text
s_k ∈ {A, P}
```

其中：

- `A` 表示远地点事件；
- `P` 表示近地点事件。

扩展后点火事件为：

```text
T_k = t_event(s_k, j_k) + delta_t_k
```

但仍建议保留以下硬约束：

```text
第 k 次点火后，最多 3 圈内必须存在远地点进入可点火经度窗口。
```

近地点点火模式可用于特殊阶段，例如强调整轨道能量、远地点高度或周期；但本任务初始规划主流程优先采用远地点点火模式。

# 22. 算法结论

本文档给出的五次脉冲点火任务规划算法具有以下特点：

1. **第一次点火由初始轨道自然传播决定**  
   先寻找最近一个满足经度窗口的远地点，作为第一次变轨的大致时间点。

2. **后续点火由远地点回归事件递推决定**  
   第 `k+1` 次点火时间由第 `k` 次点火后的远地点回归事件确定，而不是独立搜索。

3. **每次控制量都必须保证三圈内可再次点火**  
   第 `k=1..4` 次点火后，未来 3 个远地点中必须至少有一个进入可点火经度窗口。

4. **推进剂最小化在可达性约束之后进行**  
   只有满足远地点回归、经度窗口和点火时长约束的控制量，才参与推进剂最优比较。

5. **算法不使用最终优化结果作为参考**  
   所有初值和候选点均由初始轨道、动力学传播、经度窗口和任务约束自动生成。

6. **输出可直接转换为有限推力初值**  
   每个脉冲点输出 `T_k, DeltaV_k, alpha_k`，并可根据火箭方程估算有限推力的 `tb,k, ts,k, te,k`，作为后续有限推力序贯优化的起点。
