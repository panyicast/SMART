# 1. 文档目的

本文档描述一种用于五次有限推力变轨的工程化优化算法。算法面向如下任务：从给定初始高椭圆轨道出发，在考虑 J2 摄动和有限推力弧段的条件下，完成 5 次变轨，最终到达半长轴 42164.2 km、偏心率 0、轨道倾角 5°、标称定点经度 120°E 的同步轨道。

算法重点不是一次性求解全部变量的全局非线性规划，而是采用“脉冲初值规划 + 当前有限推力真实优化 + 后续剩余脉冲代价评估 + 顺序固定 + 最终整体精修”的分层策略，以提高收敛稳定性和工程可实现性。

- 约束：每次点火时间不超过 90 min。

- 约束：每次点火开始与结束时刻的星下点经度均在东经 40° 至东经 180°。

- 约束：点火期间无径向推力分量，推力方向始终位于当地水平面内。

- 控制量：每次点火开始时间、结束时间、当地水平面内点火方向角 alpha。

- 目标：总推进剂消耗最小。

## 2.3 沉底推力器与变轨发动机效率修正参数

每次主变轨发动机点火前，先进行一次沉底推力器点火。沉底推力器方向与本次主变轨点火方向一致，且同样无径向分量。变轨发动机点火期间的姿态控制推进剂消耗用效率修正参数表示，可等效为降低主变轨发动机实际比冲。以下参数均作为可配置参数处理。


| 参数 | 符号 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 沉底推力器推力 | f_set | 20 N | 可配置 |
| 沉底推力器比冲 | Isp_set | 290 s | 可配置 |
| 沉底推力器点火时长 | tau_set | 240 s | 可配置，默认每次固定 |
| 沉底推力器方向 | u_set | 与主变轨方向一致 | 无径向分量 |
| 变轨发动机姿控效率 | eta_ac | 1.73% = 0.0173 | 可配置 |
| 主发动机等效比冲 | Isp_main_eff = Isp/(1+eta_ac) | 308.758 s | 用于质量消耗 |
| 主发动机等效喷速 | c_main_eff = Isp_main_eff*g0 | 3027.886 m/s | 用于火箭方程 |
| 主发动机等效质量流率 | mdot_main_eff = f/c_main_eff | 0.161829 kg/s | 含姿控消耗 |
| 沉底推力器质量流率 | mdot_set = f_set/(Isp_set*g0) | 0.0070325 kg/s | 每次固定消耗约 1.6878 kg |


```text
Isp_main_eff = Isp_main / (1 + eta_ac)
```


```text
mdot_main_eff = f_main / (Isp_main_eff*g0) = (1+eta_ac) f_main/(Isp_main*g0)
```


```text
mdot_set = f_set/(Isp_set*g0),    m_set_per_burn = mdot_set * tau_set = 1.6878 kg
```

主变轨发动机的推力加速度仍按 f_main/m 进入动力学；效率修正只增加质量消耗，不额外增加主推力方向上的加速度。这相当于将变轨期间姿态控制消耗的推进剂并入主发动机等效比冲。

# 3. 坐标系与点火方向定义


## 3.1 状态变量


```text
X = [x, y, z, xdot, ydot, zdot, m]^T
```

位置和速度均采用地心赤道惯性坐标系 ECI。


```text
r_vec = [x, y, z]^T,    v_vec = [xdot, ydot, zdot]^T,    r = ||r_vec||
```


## 3.2 当地水平坐标系

由于点火无径向分量，推力方向必须位于当地水平面内。定义：


```text
Rhat = r_vec / ||r_vec||
```


```text
Khat = [0, 0, 1]^T
```


```text
Ehat = (Khat x Rhat) / ||Khat x Rhat||
```


```text
Nhat = [Khat - (Khat·Rhat)Rhat] / ||Khat - (Khat·Rhat)Rhat||
```


```text
Shat = -Nhat
```

其中 Ehat 为当地正东方向，Shat 为当地正南方向。


## 3.3 点火方向角 alpha

第 k 次点火方向角 alpha_k 定义为：从当地正东方向起算，向当地正南方向为正。


| alpha_k | 含义 |
| --- | --- |
| 0° | 正东方向点火 |
| 90° | 正南方向点火 |
| -90° | 正北方向点火 |
| 180° 或 -180° | 正西方向点火 |


```text
u_hat_k(t) = cos(alpha_k) Ehat(t) + sin(alpha_k) Shat(t)
```

由于 Ehat 与 Shat 均与 Rhat 正交，因此 r_vec·u_hat_k = 0，天然满足无径向推力约束。注意：alpha_k 在一次点火期间保持常值，但 Ehat(t)、Shat(t) 随卫星位置变化，所以惯性系中的推力方向不是常值。


# 4. J2 + 有限推力动力学模型


## 4.1 常数


| 参数 | 符号 | 建议值 |
| --- | --- | --- |
| 地球引力常数 | mu | 398600.4418 km³/s² |
| 地球赤道半径 | Re | 6378.137 km |
| J2 项 | J2 | 1.08262668e-3 |
| 地球自转角速度 | omega_e | 7.2921158553e-5 rad/s |


## 4.2 J2 摄动加速度


```text
a_J2 = (3 J2 mu Re^2 / 2 r^5) * [ x(5z^2/r^2 - 1), y(5z^2/r^2 - 1), z(5z^2/r^2 - 3) ]^T
```

也可按参考图写成分量形式：


```text
a_J2,x = J2 * mu*x*Re^2/r^5 * [-1.5 + 7.5(z/r)^2]
```


```text
a_J2,y = J2 * mu*y*Re^2/r^5 * [-1.5 + 7.5(z/r)^2]
```


```text
a_J2,z = J2 * mu*z*Re^2/r^5 * [-4.5 + 7.5(z/r)^2]
```


## 4.3 非点火段


```text
rddot = -mu/r^3 * r_vec + a_J2
```

mdot = 0


## 4.4 点火段


```text
rddot = -mu/r^3 * r_vec + a_J2 + (f/m) u_hat_k(t)
```


```text
mdot = -f/(Isp*g0)
```

该模型与参考图中的推力经纬度形式等价。若参考图中使用惯性推力经纬度 alpha_I、delta_I，则：


```text
alpha_I = atan2(u_y, u_x),    delta_I = asin(u_z)
```


```text
u_hat = [cos(alpha_I)cos(delta_I), sin(alpha_I)cos(delta_I), sin(delta_I)]^T
```

本文实际优化的是当地水平面内的 alpha_k，而不是参考图中的惯性推力经纬度 alpha_I、delta_I。


## 4.5 沉底推力器与主变轨发动机组合动力学

第 k 次机动由“沉底推力器预点火 + 主变轨发动机点火”两段组成。优化变量仍为主变轨发动机的开始时间 ts,k、结束时间 te,k 和方向角 alpha_k。沉底推力器开始时间由主发动机开始时间自动确定。


```text
tb,k = ts,k - tau_set
```


```text
沉底段:    t in [tb,k, ts,k]
```


```text
主变轨段:  t in [ts,k, te,k]
```

两段推力方向一致：


```text
u_hat_set,k(t) = u_hat_main,k(t) = cos(alpha_k)Ehat(t) + sin(alpha_k)Shat(t)
```

沉底推力器段动力学：


```text
rddot = -mu/r^3*r_vec + a_J2 + (f_set/m) u_hat_k(t)
```


```text
mdot = -f_set/(Isp_set*g0) = -0.0070325 kg/s
```

主变轨发动机段动力学：


```text
rddot = -mu/r^3*r_vec + a_J2 + (f_main/m) u_hat_k(t)
```


```text
mdot = -f_main/(Isp_main_eff*g0) = -0.161829 kg/s
```

非点火段质量不变。若未来需要关闭沉底推力器模型，可令 tau_set=0 或 f_set=0；若不考虑姿控效率，可令 eta_ac=0。


# 5. 星下点经度与终端轨道约束


## 5.1 星下点经度

将 ECI 位置转换到地固系 ECEF：


```text
r_ECEF = R3[-theta_G(t)] r_ECI
```


```text
lambda = atan2(y_ECEF, x_ECEF)
```

经度统一转换为 0° 至 360°，东经为正。引入沉底推力器后，第 k 次机动存在沉底开始 tb,k、主发动机开始 ts,k、主发动机结束 te,k 三个关键时刻。若经度窗口约束适用于全部有推力弧段，推荐约束：


```text
40° <= lambda(tb,k) <= 180°
```


```text
40° <= lambda(tb,k) <= 180°
```


```text
40° <= lambda(ts,k) <= 180°
```


```text
40° <= lambda(te,k) <= 180°
```

若任务明确只约束主变轨发动机，则可只检查 ts,k 与 te,k；但工程上更保守的做法是对沉底段和主变轨段的完整有推力弧段都施加经度窗口约束。若任务要求整个点火弧段都位于窗口内，则需在弧段内每 60 s 至 120 s 增加采样经度约束。


## 5.2 终端轨道参数


```text
h_vec = r_vec x v_vec
```


```text
epsilon = v^2/2 - mu/r,    a = -mu/(2 epsilon)
```


```text
e_vec = (v_vec x h_vec)/mu - r_vec/r,    e = ||e_vec||
```


```text
i = acos(h_z/||h_vec||)
```

终端偏心率为 0 时，近地点幅角存在奇异性，不建议将近地点幅角作为终端约束。建议直接约束偏心率模长 e 或偏心率矢量分量。


## 5.3 标称经度

可采用升交点赤经 Omega 与纬度幅角 u 构造标称经度：


```text
n_vec = Khat x h_vec
```


```text
Omega = atan2(n_y, n_x)
```


```text
u = atan2(r_vec·q_hat, r_vec·n_hat),    q_hat = h_hat x n_hat
```


```text
lambda_nom = Omega + u - theta_G(t)
```


```text
wrap180(lambda_nom - 120°) = 0
```


# 6. 总体优化思想

直接一次性优化 5 次有限推力点火的 15 个变量，容易因 J2 摄动、经度窗口、长弧段有限推力和终端同步轨道约束而不收敛。为降低难度，采用序贯滚动优化方法。

- 先用脉冲法求五次变轨的可行初值。

- 按第 1 次到第 5 次逐次优化有限推力点火参数。

- 第 k 次优化时，前 k-1 次有限推力点火已经固定。

- 第 k 次为真实有限推力积分。

- 第 k+1 至第 5 次用剩余脉冲规划器估计后续最小代价。

- 当前目标函数 = 本次沉底推力器消耗 + 本次主变轨发动机等效消耗 + 后续剩余脉冲等效消耗。

- 五次全部优化后，再以序贯解为初值释放全部 15 个变量整体精修。


# 7. 五次脉冲初始规划


## 7.1 脉冲规划变量


```text
z = [T1, DeltaV1, alpha1, ..., T5, DeltaV5, alpha5]
```

脉冲速度更新为：


```text
v_k^+ = v_k^- + DeltaV_k [cos(alpha_k)Ehat_k + sin(alpha_k)Shat_k]
```

脉冲之间按 J2 摄动传播。


## 7.2 脉冲规划目标与约束


```text
min sum(DeltaV_k),    k=1..5
```


```text
40° <= lambda(T_k) <= 180°
```


```text
T_{k+1} > T_k
```


```text
af = 42164.2 km,    ef = 0,    if = 5°,    lambda_nom,f = 120°E
```


## 7.3 脉冲到有限推力初值转换

引入沉底推力器后，第 k 次脉冲速度增量 DeltaV_k 应转换为“沉底推力器预点火 + 主发动机点火”的组合初值。设本次机动前质量为 m_b,k。


```text
m_s,k = m_b,k - mdot_set * tau_set
```


```text
DeltaV_set,k ≈ c_set * ln(m_b,k / m_s,k),    c_set = Isp_set*g0
```


```text
DeltaV_main,k = max(0, DeltaV_k^0 - DeltaV_set,k)
```


```text
tau_main,k^0 = (m_s,k/mdot_main_eff) [1 - exp(-DeltaV_main,k/c_main_eff)]
```

默认将主变轨发动机点火弧段的中点对齐到脉冲时刻 T_k^0，因为主发动机提供绝大部分速度增量。


```text
ts,k^0 = T_k^0 - tau_main,k^0/2
```


```text
te,k^0 = T_k^0 + tau_main,k^0/2
```


```text
tb,k^0 = ts,k^0 - tau_set
```

方向角初值取脉冲方向角 alpha_k^0。若希望更严格，可按组合推力弧段的冲量质心而不是主发动机中点对齐 T_k^0。

# 8. 有限推力序贯优化子问题


## 8.1 当前点火优化变量


```text
p_k = [ts,k, te,k, alpha_k]
```

也可使用中心时刻和持续时间形式，提高数值稳定性：


```text
p_k = [c_k, tau_k, alpha_k]
```


```text
c_k = (ts,k + te,k)/2,    tau_k = te,k - ts,k
```


```text
ts,k = c_k - tau_k/2,    te,k = c_k + tau_k/2
```


## 8.2 当前机动推进剂消耗

第 k 次机动的推进剂消耗由沉底推力器和主变轨发动机两部分组成。定义主发动机点火时长：


```text
tau_main,k = te,k - ts,k
```


```text
m_p,set,k = mdot_set * tau_set
```


```text
m_p,main,k = mdot_main_eff * tau_main,k
```


```text
m_p,k^finite = m_p,set,k + m_p,main,k
```


```text
m_p,k^finite = 0.0070325*240 + 0.161829*(te,k-ts,k)
```

其中 tau_main,k 单位为秒。若 tau_set、f_set、Isp_set 或 eta_ac 改变，上式自动随参数更新。


## 8.3 第 k 次子问题目标函数

第 k 次有限推力结束后，得到状态 X_e,k、质量 m_e,k、时刻 te,k。若 k < 5，则从该状态出发，用剩余 5-k 次脉冲重新规划，得到剩余最小总速度增量 DeltaV_rem,k*。


```text
DeltaV_rem,k* = min sum_{j=k+1..5} DeltaV_j
```

将剩余脉冲速度增量转换为等效推进剂消耗：


```text
m_p,rem,k = rem_propellant_equivalent({DeltaV_j}, m_e,k, N_rem, tau_set, c_set, c_main_eff)
```

因此第 k 次子问题目标函数为：


```text
J_k = m_p,set,k + m_p,main,k + m_p,rem,k
```

当 k = 5 时，已经没有剩余脉冲，目标函数改为本次沉底推力器消耗 + 本次主变轨发动机等效消耗 + 终端轨道偏差罚函数，或直接采用硬终端约束。

推荐的剩余脉冲推进剂等效转换按每个剩余脉冲逐次计算：每个未来脉冲前先扣除一次沉底推力器消耗，并估算沉底段提供的微小 DeltaV，再用主发动机等效比冲计算剩余 DeltaV 所需推进剂。快速近似时，也可使用 N_rem*m_p,set + m_e,k[1-exp(-DeltaV_rem,k*/c_main_eff)]，但精度略低。

```text
0 < te,k - ts,k <= 5400 s    # 若 90 min 只限制主发动机
```


```text
tb,k = ts,k - tau_set,    tb,k > te,k-1
```


```text
40° <= lambda(tb,k) <= 180°
```


```text
40° <= lambda(ts,k) <= 180°
```


```text
40° <= lambda(te,k) <= 180°
```


```text

```

角度变量需要做 wrap 处理，避免 179° 与 -179° 附近出现数值跳变。


# 9. 剩余脉冲余量评估器

剩余脉冲评估器是外层有限推力优化的核心。它用于回答：在当前一次有限推力候选解执行完以后，剩下的机动次数是否还能完成终端目标？如果可以，最小还需要多少脉冲 DeltaV？


## 9.1 输入与输出


| 类别 | 内容 |
| --- | --- |
| 输入 | 当前状态 X_c、当前时刻 t_c、剩余脉冲次数 N_rem、经度窗口、终端目标 |
| 输出 | 剩余脉冲时刻 T_j、速度增量 DeltaV_j、方向角 alpha_j、总 DeltaV、可行性标志 |


## 9.2 剩余脉冲优化问题


```text
min sum_{j=1..N_rem} DeltaV_j
```


```text
DeltaV_j >= 0
```


```text
T_{j+1} > T_j
```


```text
40° <= lambda(T_j) <= 180°
```


```text
af = 42164.2 km, ef = 0, if = 5°, lambda_nom,f = 120°E
```


## 9.3 安全裕度

由于剩余脉冲最终仍需转换为有限推力弧段，建议内层脉冲规划时采用收缩经度窗口，避免脉冲点贴近边界。


```text
45° <= lambda(T_j) <= 175°
```

或更保守地使用 50° 至 170°。这样后续展开为有限推力时，点火开始和结束点更容易满足原始 40° 至 180° 约束。


# 10. 第 1 至第 5 次点火优化流程


## 10.1 第一次点火

第一次点火优化变量为 [ts,1, te,1, alpha1]。候选解评估流程如下：

- 从初始状态无推力传播到 ts,1。

- 检查 ts,1 星下点经度。

- 在 tb,1 至 ts,1 进行沉底推力器有限推力积分；在 ts,1 至 te,1 进行主变轨发动机有限推力积分。

- 检查 te,1 星下点经度。

- 得到第一次点火结束状态 X_e,1、m_e,1。

- 从该状态出发，用后四次脉冲重新规划。

- 目标函数 J1 = 本次沉底消耗 + 本次主变轨等效消耗 + 后四次脉冲等效消耗。


```text
J1 = m_p,set,1 + m_p,main,1 + m_p,rem,1
```

优化完成后固定第一次点火参数。


## 10.2 第二至第四次点火

第二至第四次点火完全采用同样形式，只是剩余脉冲次数依次减少。


| 阶段 | 优化变量 | 剩余任务 | 目标函数 |
| --- | --- | --- | --- |
| 第 2 次 | [ts,2, te,2, alpha2] | 后 3 次脉冲 | J2 = 本次沉底 + 主变轨消耗 + 后 3 次脉冲等效消耗 |
| 第 3 次 | [ts,3, te,3, alpha3] | 后 2 次脉冲 | J3 = 本次沉底 + 主变轨消耗 + 后 2 次脉冲等效消耗 |
| 第 4 次 | [ts,4, te,4, alpha4] | 后 1 次脉冲 | J4 = 本次沉底 + 主变轨消耗 + 后 1 次脉冲等效消耗 |


## 10.3 第五次点火

第五次点火后没有剩余脉冲，因此必须直接满足终端轨道约束。


```text
min [m_p,set,5 + mdot_main_eff*(te,5-ts,5)]
```


```text
subject to: af=42164.2 km, ef=0, if=5°, lambda_nom,f=120°E
```

如果使用罚函数形式，则：


```text
J5 = m_p,set,5 + mdot_main_eff*(te,5-ts,5) + P_f
```

其中 P_f 为半长轴、偏心率、倾角和标称经度误差的归一化二次罚函数。


# 11. 最终 15 变量整体局部精修

序贯优化是贪心过程，早期点火一旦固定，后续无法反向修正。因此五次点火全部得到后，应以序贯结果为初值，释放全部 15 个变量做整体局部精修。


```text
P = [ts,1, te,1, alpha1, ..., ts,5, te,5, alpha5]
```


```text
min J = sum_{k=1..5}[mdot_set*tau_set + mdot_main_eff*(te,k-ts,k)]
```

整体精修的约束包括：每次点火时长、点火顺序、点火起止经度、终端轨道参数。


| 约束类型 | 表达式 |
| --- | --- |
| 点火时长 | 0 < te,k - ts,k <= 5400 s；若按完整有推力段限制，则 tau_set + te,k - ts,k <= 5400 s |
| 点火顺序 | tb,k+1 = ts,k+1 - tau_set > te,k |
| 沉底开始经度 | 40° <= lambda(tb,k) <= 180° |
| 主发动机开始经度 | 40° <= lambda(ts,k) <= 180° |
| 主发动机结束经度 | 40° <= lambda(te,k) <= 180° |
| 终端半长轴 | af = 42164.2 km |
| 终端偏心率 | ef = 0 |
| 终端倾角 | if = 5° |
| 终端标称经度 | lambda_nom,f = 120°E |


# 12. 伪代码


```text
输入：
    初始轨道根数 OE0、初始时刻 t0、初始质量 m0
    主发动机推力 f_main、比冲 Isp_main、姿控效率 eta_ac、最大单次点火时长 5400 s
    沉底推力器 f_set、Isp_set、tau_set
    经度窗口 [40E, 180E]
    终端目标 a=42164.2 km, e=0, i=5 deg, lon=120E

预处理：
    OE0 -> ECI 状态 X0
    c_main_eff = Isp_main*g0/(1+eta_ac)
    mdot_main_eff = f_main / c_main_eff
    c_set = Isp_set*g0
    mdot_set = f_set / c_set

步骤 0：五次脉冲初始规划
    pulse_plan = solve_impulse_plan(X0, t0, N=5)
    若不可行：调整初值、任务时间范围或脉冲搜索策略

步骤 1：序贯有限推力优化
    fixed_burns = []

    for k = 1 to 5:
        根据 fixed_burns 从 X0,m0,t0 正向积分，得到第 k 次优化前状态 X_pre,m_pre,t_pre

        根据当前脉冲方案生成初值：
            m_s = m_pre - mdot_set*tau_set
            DeltaV_set = c_set*ln(m_pre/m_s)
            DeltaV_main = max(0, DeltaV_k - DeltaV_set)
            tau0 = m_s/mdot_main_eff * (1 - exp(-DeltaV_main/c_main_eff))
            ts0 = T_k - tau0/2
            te0 = T_k + tau0/2
            tb0 = ts0 - tau_set
            alpha0 = alpha_k

        定义 objective_k(ts, te, alpha):
            若 te <= ts 或 te-ts > 5400：return penalty
            tb = ts - tau_set
            若按完整有推力段限制且 te-tb > 5400：return penalty
            若 tb <= 上一次点火结束时刻：return penalty

            无推力传播至 tb，检查 lambda(tb)
            在 [tb,ts] 内进行沉底推力器积分：
                u = cos(alpha)*Ehat + sin(alpha)*Shat
                rddot = -mu/r^3*r + a_J2 + f_set/m*u
                mdot = -f_set/(Isp_set*g0)
            检查 lambda(ts)
            在 [ts,te] 内进行主变轨发动机积分：
                每一步计算 Ehat, Shat
                u = cos(alpha)*Ehat + sin(alpha)*Shat
                rddot = -mu/r^3*r + a_J2 + f_main/m*u
                mdot = -f_main/(Isp_main_eff*g0)
            检查 lambda(te)

            得到 Xe, me, te

            if k < 5:
                rem_plan = solve_impulse_plan(Xe, te, N=5-k)
                若不可行：return penalty
                DeltaV_rem = sum(rem_plan.DeltaV)
                J = mdot_set*tau_set + mdot_main_eff*(te-ts) + rem_propellant_equivalent(rem_plan, me)
            else:
                计算终端轨道 af, ef, if, lonf
                J = mdot_set*tau_set + mdot_main_eff*(te-ts) + terminal_penalty(af, ef, if, lonf)
            return J

        [ts*, te*, alpha*] = optimize(objective_k, 初值=[ts0, te0, alpha0])
        fixed_burns.append([ts*, te*, alpha*])

        从 X0,m0,t0 按 fixed_burns 正向积分，得到第 k 次结束状态
        if k < 5:
            pulse_plan = solve_impulse_plan(X_after, t_after, N=5-k)

步骤 2：整体局部精修
    以 fixed_burns 为初值，释放全部 15 个变量
    求解完整有限推力优化问题

输出：
    五次点火开始时间、结束时间、方向角、消耗、经度、终端轨道误差
```

# 15. 收敛问题与处理方法


## 15.1 剩余脉冲规划不可行

- 可能原因：当前有限推力候选解把轨道推入不利状态，剩余次数不足，或经度窗口过窄。

- 处理方法：对该候选解加大罚值；增加脉冲多初值；适当延长任务时间窗口；内层脉冲先用宽松窗口再收紧。


## 15.2 第五次难以满足终端偏心率

- 可能原因：前四次固定后剩余调节能力不足，第五次点火位置偏离圆化点，或第五次时长不足。

- 处理方法：第四次优化时对第五次剩余脉冲位置留裕度；第五次做多初值搜索；最后进行 15 变量整体精修。


## 15.3 起止经度满足但弧段中间越界

- 若任务要求整个点火弧段都在窗口内，必须增加弧段内部采样经度约束。

- 建议每隔 60 s 检查一次经度，任一点越界则返回罚值。


## 15.4 局部最优

- 对点火中心时刻在脉冲中心前后 ±30 min、±60 min 做多初值。

- 对方向角在初值附近 ±10°、±20°、±40° 做多初值。

- 先使用粗搜索，再使用 SQP/SNOPT/IPOPT 精修。


# 16. 结论

本文档给出的算法将复杂的五次有限推力轨道优化问题拆解为一系列低维子问题。每次只优化当前主变轨发动机的开始时间、结束时间和当地水平面方向角，沉底推力器点火由主机开始时间自动前推 240 s 得到，同时利用剩余脉冲规划器估计后续任务代价。该方法不是严格全局最优方法，但具有很强的工程实用性和收敛稳定性。

推荐的完整流程为：五次脉冲全局初解 → 第 1 次有限推力优化 + 后 4 次脉冲评估 → 固定第 1 次 → 依次优化第 2 至第 5 次 → 最终释放全部 15 个变量整体局部精修 → 输出最终点火策略和约束验证报告。
